# Per-Store flat-fee subscription with code-defined Plans and Entitlements

One Stripe **Subscription** per **Store**, billed via the **AccountOwner** **Employee**. Three **Plan** tiers — Starter / Growth / Pro — gated by feature keys, not seat count, so adding a teammate does not change the bill. **Entitlements** are a read-side cache (Store → feature keys) rebuilt by the Stripe webhook handler; the rest of the app reads Entitlements and never talks to Stripe directly, which means feature gates work offline and in tests without a Stripe round-trip. Entitlements and **Permissions** are independent layers: a feature is accessible only if the Store has the Entitlement AND the User has the Permission. Conflating the two would mean either every Plan tier needs its own permission set or every permission needs a per-tier override — both are worse. **AccountOwner** is reassignable without re-creating the Stripe customer (we update the customer's metadata and contact instead).
