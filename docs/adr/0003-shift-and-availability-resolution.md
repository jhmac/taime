# Shift is the canonical word; availability resolution is layered

A row in the `schedules` table is a **Shift** — singular, one Employee, one Location, start and end. The word "Schedule" is reserved for the read-side collection of Shifts for a Store + Location + week, and is never a row. **Availability** is resolved top-down with the highest-precedence layer winning: approved **TimeOffRequest** > **ManagerOverride** > **EmployeeOverride** > **AvailabilityTemplate** > unavailable. Defaulting to unavailable (rather than available) prevents the AI auto-scheduler from silently scheduling Employees who never confirmed they could work. The legacy `userAvailability` per-payroll-period submission table is deprecated in favor of templates + overrides; it remains in the schema for backfill but no new feature should write to it.
