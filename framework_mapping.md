# Taime - Framework & Feature Mapping

## Frontend
| Framework/Library | Purpose | Linked Features |
| :--- | :--- | :--- |
| **React** | UI Library | Core component architecture, reactive user interface. |
| **TypeScript** | Type Safety | Universal type safety across the entire application. |
| **Vite** | Build Tool | Development server, fast HMR, and production bundling. |
| **Tailwind CSS** | Styling | Utility-first CSS for responsive, mobile-first design. |
| **shadcn/ui** | UI Components | Accessible, themed components (buttons, dialogs, forms). |
| **TanStack Query** | State Management | Server state handling, caching, and data fetching (v5). |
| **Wouter** | Routing | Lightweight client-side navigation. |
| **Lucide React** | Icons | Consistent iconography for actions and visual cues. |
| **Recharts** | Data Vis | Visual Analytics Dashboard (labor costs, punctuality). |
| **Leaflet** | Maps | Interactive geofencing boundary drawing. |
| **date-fns** | Date Logic | Scheduling, time tracking, and holiday pay calculations. |
| **React Hook Form** | Form Handling | Robust form validation and state (with Zod). |

## Backend
| Framework/Library | Purpose | Linked Features |
| :--- | :--- | :--- |
| **Node.js / Express** | Server | API routing, middleware, and business logic. |
| **Drizzle ORM** | Database Layer | Type-safe PostgreSQL interactions. |
| **Neon / PostgreSQL** | Database | Persistent storage for users, shifts, and settings. |
| **Clerk** | Authentication | User login, SSO, and Role-Based Access Control (RBAC). |
| **WebSockets (ws)** | Real-time | Team messaging, live geofence monitoring, notifications. |
| **Zod** | Validation | Runtime type checking for API requests and schema. |
| **Helmet / Rate Limit**| Security | HTTP header security and API rate limiting. |

## AI & Integrations
| Service/API | Purpose | Linked Features |
| :--- | :--- | :--- |
| **Anthropic Claude** | AI Engine | Auto-scheduling, task assignment, Success Assistant. |
| **Shopify API** | Integration | Sales data sync for staffing recommendations. |
| **Nylas API** | Communications | Automated team invitation emails. |
| **Web Push API** | Notifications | Browser/Mobile push alerts for schedule changes. |
