Build the full Issue Tracker feature — API endpoints and frontend UI.

BACKEND — Create `server/routes/issues.ts`:

1. POST /api/issues — Create a new issue
   - Auth: Any authenticated employee
   - Zod body: { title, description?, category, priority?, photo_url?, related_sop_id? }
   - Auto-set: store_id from auth, reported_by from auth, status = 'open'
   - Notify managers via WebSocket: { type: 'issue_created', data: { issue } }
   - If priority is 'urgent', also notify the owner
   - Log to activity_log

2. GET /api/issues — List issues
   - Auth: Any authenticated employee
   - Query params: status?, category?, priority?, assigned_to?, date_from?, date_to?, search?
   - Paginated
   - Sort by: priority (urgent first), then created_at desc

3. GET /api/issues/:id — Get issue detail with comments
   - Auth: Any authenticated employee

4. PUT /api/issues/:id — Update issue
   - Auth: Manager/Owner for assignment and status changes, any employee for adding details
   - Zod body: { title?, description?, category?, priority?, status?, assigned_to?, resolution_notes? }
   - If status changes to 'resolved': set resolved_at and resolved_by
   - Notify relevant parties via WebSocket
   - Log status changes to activity_log

5. POST /api/issues/:id/comments — Add a comment
   - Auth: Any authenticated employee
   - Zod body: { comment_text }
   - Notify the issue reporter and assignee via WebSocket

FRONTEND — Create components in `client/src/features/issues/`:

1. IssueListPage (`/issues`)
   - Kanban-style columns: Open | In Progress | Waiting | Resolved
   - Or toggle to list view (table with sortable columns)
   - Filter bar: category, priority, assigned to
   - Each issue card: title, category badge, priority indicator (color-coded), reporter name, age (e.g., "2h ago")
   - "Report Issue" floating action button (always visible, big and obvious)

2. ReportIssueDialog (modal/sheet)
   - Dead simple form — must be completable in under 30 seconds:
     - Title (required, text input with placeholder: "What's the problem?")
     - Category (required, icon-based selection grid — not a dropdown)
     - Priority (optional, defaults to medium, visual selector: 🟢 Low | 🟡 Medium | 🔴 High | 🚨 Urgent)
     - Description (optional textarea)
     - Photo (optional camera/upload button)
   - "Report" button → submit and close
   - Success toast: "Issue reported! Your manager has been notified."

3. IssueDetailPage (`/issues/:id`)
   - Full issue details
   - Status workflow buttons (Manager/Owner): Open → In Progress → Waiting/Resolved → Closed
   - Assignment dropdown (Manager/Owner)
   - Comment thread
   - If there's a related SOP, show a link: "Related Procedure: [SOP Title]"
   - Resolution notes field (shown when resolving)

DESIGN:
- The "Report Issue" button should be accessible from ANY page in the app (add it to the main layout)
- Mobile-first: large tap targets, minimal typing required
- Priority colors: low=green, medium=yellow, high=red, urgent=pulsing red
- Celebration when resolved: "Issue resolved! Thanks for keeping things running smooth."
