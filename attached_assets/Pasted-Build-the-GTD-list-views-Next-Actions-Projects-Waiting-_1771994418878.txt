Build the GTD list views — Next Actions, Projects, Waiting For, and Someday/Maybe.

LOCATION: `client/src/features/gtd/`

1. NEXT ACTIONS PAGE (`/gtd/actions`)
   
   This is THE daily work view. "What should I do right now?"
   
   CONTEXT-BASED FILTERING (the GTD killer feature):
   - Top filter bar with context chips: All | @Store | @Computer | @Phone | @Errands | @Anywhere
   - Secondary filters: Energy (Low/Med/High), Time Available (< 5 min, < 15 min, < 30 min, < 1 hr, any)
   - These filters answer: "I'm at the store, I have 15 minutes and low energy — what can I do?"
   
   Layout:
   - "Two-Minute Actions" section at top (if any exist) — highlighted with ⚡ icon and warm yellow background
     - These are meant to be knocked out immediately
   - "Due Today" section — actions due today, red if overdue
   - "Active Actions" — everything else, grouped by priority
   - Each action card:
     - Checkbox (tap to complete — celebratory animation)
     - Title
     - Context badge (@store, @phone, etc.)
     - Time estimate if set
     - Priority indicator (color-coded dot)
     - Project link if associated
     - Due date if set (red if overdue, yellow if due today)
   - Completing an action: checkbox animation → brief celebration toast → card slides away
   - Swipe actions: swipe right to complete, swipe left to defer/edit

2. PROJECTS PAGE (`/gtd/projects`)
   
   Layout:
   - List of active projects
   - Each project card:
     - Title
     - Progress bar (completed actions / total actions)
     - Next action preview (the most important next step)
     - Due date if set
     - Owner avatar
   - Tap project → ProjectDetailPage

3. PROJECT DETAIL PAGE (`/gtd/projects/:id`)
   
   Layout:
   - Project title, description, desired outcome
   - Progress ring/bar
   - "Next Actions" list (linked actions for this project) — can add new ones inline
   - "Waiting For" list (things blocked on others)
   - "Reference" section (supporting material)
   - "Complete Project" button (when all actions are done)
   - Completion celebration: "Project complete! [title] is done! 🎉"

4. WAITING FOR PAGE (`/gtd/waiting`)
   
   Layout:
   - List of items you're waiting on
   - Each card: description, who you're waiting on, follow-up date
   - OVERDUE HIGHLIGHT: items past their follow-up date show in red with "Follow up needed!"
   - Tap to mark as received or update follow-up date
   - "Nudge" button for items with a waiting_on_employee_id — sends a gentle WebSocket notification

5. SOMEDAY/MAYBE PAGE (`/gtd/someday`)
   
   Layout:
   - The "guilt-free parking lot" for ideas
   - Grouped by category (or ungrouped)
   - Each card: title, description, category badge
   - "Activate" button → converts to a Project or Next Action
   - Warm messaging: "These are your seeds. When the time is right, plant one. 🌱"

6. GTD NAVIGATION:
   - Add a GTD section to the main app navigation
   - Tab bar or sidebar items: Inbox (with count badge), Actions, Projects, Waiting, Someday
   - The count badges show: Inbox unprocessed count, overdue actions count, overdue follow-ups count

7. GTD DASHBOARD WIDGET:
   - Add to the existing role-based dashboards (all roles):
   - Compact card showing: Inbox count, Actions due today, Overdue items, Two-minute quick wins available
   - "What should I do next?" button → takes you to the context-filtered actions view

DESIGN:
- The GTD system should feel CALM, not stressful. Clean, organized, minimal.
- Completing items feels satisfying (animations, celebrations)
- Overdue items are prominent but not anxiety-inducing (gentle red, not alarming)
- Someday/Maybe feels like a creative garden, not a failure dump
- Context-based filtering is the star — it's what makes GTD work in practice

TECHNICAL:
- React Query for all data fetching with appropriate stale times
- Optimistic updates for completing actions and processing items
- Drag-and-drop for reordering actions within a project (optional, nice-to-have)
- Keyboard shortcuts on desktop: 'c' to capture, Enter to process, Space to complete
- TypeScript strict: define all GTD interfaces
