# Addition to ELON Prompt — Add This Section to the Existing ELON with Playwright Prompt

## Add: ELON Strategic Test Journal (ELON-REPORT.md)

After every ELON cycle (crawl + analysis + fixes + verification), ELON should generate and update a markdown file called `ELON-REPORT.md` in the project root. This is ELON’s memory — it reads this file at the start of every cycle so it knows what’s been tested, what passed, what failed, and what still needs work.

### How to Implement

**1. Add `_writeElonReport()` function to `elon.js`:**

After every ELON cycle (both `runElonCycle` and `evaluateConstraint`), call a function that reads the existing ELON-REPORT.md (if it exists), merges in the new data, and writes the updated file. The function should:

- Load previous report data from `.apppilot/elon-report-data.json` (structured data)
- Merge new crawl results, constraint status, and cycle results
- Calculate goals progress by cross-referencing constraints with GOALS.md priorities
- Calculate quality targets vs actual from the latest crawl metrics
- Generate the markdown from the structured data
- Write both the JSON (for memory) and the MD (for humans) files

```javascript
function _writeElonReport(projectRoot, dataDir, reportData) {
  // Save structured data (ELON reads this as memory)
  fs.writeFileSync(
    path.join(dataDir, 'elon-report-data.json'),
    JSON.stringify(reportData, null, 2)
  );
  
  // Generate readable markdown (humans read this)
  const md = _generateReportMarkdown(reportData);
  fs.writeFileSync(path.join(projectRoot, 'ELON-REPORT.md'), md);
}
```

**2. ELON reads its own report at the start of every cycle:**

In `runElonCycle`, before asking Claude to identify the limiting factor, load the previous report:

```javascript
const previousReport = _loadElonReport(dataDir);
```

Add this to the task payload sent to Claude:

```javascript
const task = {
  // ... existing fields ...
  previousReport: previousReport ? {
    constraintsSolved: previousReport.constraintsSolved,
    activeConstraint: previousReport.activeConstraint,
    crawlHistory: previousReport.crawlHistory.slice(-3), // last 3 crawls
    issuesTrending: previousReport.issuesTrending, // issues getting worse/better
    goalsProgress: previousReport.goalsProgress,
    qualityTargetsStatus: previousReport.qualityTargets,
    failedAttempts: previousReport.failedAttempts, // things ELON tried that didn't work
  } : null,
};
```

**3. Update the ELON subagent template to use the report:**

Add to `templates/subagents/elon.md`:

```
## Your Memory (Previous Report)

You have a report from previous cycles. USE IT:
- Don't re-identify constraints you've already solved
- Don't repeat approaches that failed (check failedAttempts)
- Prioritize issues that are trending WORSE (getting more severe over time)
- Focus on constraints that block the most goals (check goalsProgress)
- Check quality targets — fix metrics that are furthest from their target first
- If a constraint was partially resolved, pick up where you left off
```

**4. Track trending issues:**

In the report data, track how issues change over time:

```javascript
// After each crawl, compare with previous crawl
for (const issue of currentIssues) {
  const previous = previousIssues.find(p => p.type === issue.type && p.url === issue.url);
  if (previous) {
    issue.trend = issue.severity > previous.severity ? 'worsening' 
                : issue.severity < previous.severity ? 'improving'
                : 'stable';
    issue.firstSeen = previous.firstSeen;
    issue.cyclesSeen = (previous.cyclesSeen || 1) + 1;
  } else {
    issue.trend = 'new';
    issue.firstSeen = new Date().toISOString();
    issue.cyclesSeen = 1;
  }
}
```

**5. Track failed attempts:**

When a spec fails or a constraint isn’t resolved after a full attempt, log it:

```javascript
if (evalResult.status !== 'constraint-resolved') {
  reportData.failedAttempts.push({
    constraint: constraint.description,
    approach: constraint.steps.map(s => s.description),
    reason: evalResult.reason || 'verification-failed',
    timestamp: new Date().toISOString(),
  });
}
```

This prevents ELON from trying the same approach twice.

**6. Add report to dashboard:**

Add a “📊 View Report” button/tab in the dashboard that renders ELON-REPORT.md. Also add API endpoint:

```
GET /apppilot/api/elon/report — returns the structured report data as JSON
```

The dashboard should show:

- Goals progress bars
- Quality targets gauge chart (green/yellow/red)
- Constraint leaderboard table
- Crawl results timeline
- Trending issues (getting worse highlighted in red)
