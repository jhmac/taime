Build the SOP Execution UI — this is what employees see when they're running through a checklist.

LOCATION: Add to `client/src/features/sops/`

THE EXECUTION EXPERIENCE:

1. SOPExecutionPage (`/sops/execute/:executionId`)
   - Full-screen, mobile-optimized "checklist runner" view
   - Shows one step at a time (like a wizard/stepper), NOT a scrollable list
   - Top bar: SOP title, progress indicator (step 3 of 12), elapsed time
   - Current step card:
     - Step title (large, bold)
     - Step description (if present)
     - Step type indicator icon
     - Training mode toggle — if ON, show the training_detail text below the description in a highlighted callout box
     - Action area based on step type:
       - Action: "Mark Complete" button
       - Verification: "Confirm Done" button with a checkbox feel
       - Photo: Camera capture button (use navigator.mediaDevices.getUserMedia or file input for mobile) → shows preview → "Upload & Complete"
       - Decision: Show option buttons, selecting one advances to the appropriate step
       - Timer: Countdown timer UI with start button, auto-completes when timer reaches zero
     - "Skip" button (secondary/ghost style) — if clicked, show a text input for skip reason (required) then confirm
     - If the step is a checkpoint: show a badge "Manager Sign-off Required" — after completing, show "Waiting for manager approval" state
   - Navigation: Back button (to revisit previous steps, read-only), step dots for progress
   - Bottom: Notes input (optional, collapses into an icon when empty)

2. When all steps are done:
   - Show a celebration screen: "🎉 [SOP Title] Complete! Nice work, [employee name]!"
   - Show completion stats: total time, steps completed vs skipped
   - "What bugged you about this process?" prompt (free text, optional — this feeds into the improvement engine later)
   - "Done" button → returns to dashboard

3. Checkpoint sign-off flow (for managers):
   - When a step needs sign-off, the manager gets a WebSocket notification
   - Manager can view the step, see the photo if present, and tap "Approve" or "Request Redo"
   - Create a small `SOPSignOffDialog` component that can be triggered from the notification

DESIGN PRINCIPLES:
- This is the most-used screen in the entire app. It must be FAST and SIMPLE.
- Big tap targets (minimum 44px) — employees are using this with one hand in a busy store
- Minimal visual clutter — one step at a time, clear action
- Celebration over surveillance: completion messages are warm and encouraging
- Progress feels satisfying — animate the progress bar, use subtle haptic feedback cues (CSS animations)
- Works offline-ish: if the network drops mid-execution, queue step completions locally and sync when reconnected

TECHNICAL:
- Use the execution API endpoints from the previous prompt
- Real-time updates via WebSocket for sign-off status changes
- Photo upload: compress images client-side before upload (max 1MB) — use canvas resize
- Store execution state in React state + persist to localStorage as backup
- TypeScript strict: no `any` types
