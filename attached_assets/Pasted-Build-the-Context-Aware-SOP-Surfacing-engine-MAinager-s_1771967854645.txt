Build the Context-Aware SOP Surfacing engine — MAinager's killer feature. SOPs automatically appear for the right person at the right time.

BACKEND — Create `server/services/sopSurfacing.ts`:

This service runs background checks and triggers SOP surfacing via WebSocket push and API responses.

SURFACING TRIGGERS TO IMPLEMENT:

1. TIME-BASED SURFACING:
   - When an employee clocks in for the FIRST shift of the day: surface the Opening Checklist SOP
   - When it's 30 minutes before store close: surface the Closing Checklist SOP for whoever is on shift
   - Implementation: Create a cron job that runs every 5 minutes, checks the schedule + store hours, and pushes WebSocket events
   - WebSocket event: { type: 'sop_surfaced', data: { template_id, reason: 'time_based', message: 'Time to start the opening checklist!' } }

2. EVENT-BASED SURFACING (Shift Handoff):
   - When a new employee clocks in while another employee is still on shift: surface the Shift Handoff SOP for both employees
   - Detection: in the clock-in endpoint, check if there are active time_entries for other employees at the same store
   - WebSocket event to both employees

3. ROLE-BASED SURFACING:
   - Track employee "tenure" (how long they've been at the store or how many times they've completed a specific SOP)
   - If an employee has completed an SOP fewer than 3 times: auto-enable training mode when surfacing that SOP
   - Implementation: query sop_executions count per template per employee

4. ISSUE-BASED SURFACING:
   - When a new issue is created: check if any SOP has a matching category
   - If a match is found, include the SOP link in the issue notification
   - Example: issue category 'equipment' + SOP category 'safety' or matching title keywords

5. API ENDPOINT for manual surfacing:
   - GET /api/sops/surfaced — Returns SOPs that should be surfaced RIGHT NOW for the authenticated employee
   - Checks: current time vs store hours, active shift status, role, recent executions, open issues
   - Returns array of { template_id, title, reason, priority, training_mode_recommended }
   - The frontend dashboard polls this endpoint or receives WebSocket pushes

FRONTEND:

6. SurfacedSOPBanner component:
   - Appears at the top of the employee dashboard when an SOP is surfaced
   - Shows: SOP title, reason it's surfaced, "Start Now" button
   - Dismissible (but logs the dismissal)
   - If multiple SOPs are surfaced, stack them with the highest priority on top
   - Warm messaging: "Opening time! Your Opening Checklist is ready." not "SOP #47 requires execution."

7. Integration with existing clock-in flow:
   - After a successful clock-in, check for surfaced SOPs and show the banner immediately

TECHNICAL:
- The cron job for time-based surfacing should be lightweight — query only active stores with employees on shift
- Use the cache utility for store hours lookups
- All WebSocket events follow the existing event format
- Structured logging for every surfacing trigger (which SOP, which employee, which trigger, timestamp)
