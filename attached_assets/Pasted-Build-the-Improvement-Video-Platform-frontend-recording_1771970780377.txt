Build the Improvement Video Platform frontend — recording, feed, and social features.

COMPONENTS in `client/src/features/videos/`:

1. ImprovementFeedPage (`/improvements`)
   - YouTube-style grid layout:
     - Thumbnail cards in a responsive grid (2 columns mobile, 3 tablet, 4 desktop)
     - Each card: video thumbnail (or a placeholder with play icon), title, employee name + avatar, time ago, like count, comment count
     - Click card → opens VideoPlayerModal
   - Top section: "Improvement of the Week" featured banner (if any video has is_featured = true)
   - Filter tabs: All | Most Recent | Most Liked | By Category
   - Employee filter dropdown
   - "Record Improvement" floating action button (prominent, always visible)

2. VideoRecordDialog (modal/sheet)
   - Step 1: Camera view
     - Use navigator.mediaDevices.getUserMedia({ video: true, audio: true })
     - Show live camera preview
     - Record button (large, red, centered at bottom)
     - During recording: show elapsed time counter (max 60 seconds)
     - Auto-stop at 60 seconds
     - Stop button to end early
     - After recording: show preview with Play/Re-record/Use This options
   - Step 2: Details
     - Title input (required): "What did you improve?"
     - Description textarea (optional): "Tell us about the before and after"
     - Category selector (icon grid, same categories as issues)
   - Step 3: Upload
     - Show upload progress bar
     - Get presigned URL → upload video → create video record
     - On success: "Your improvement is live! 🎉" with link to view it
   - Fallback for browsers without camera access:
     - Show a file upload input instead
     - Accept video/mp4 and video/quicktime

3. VideoPlayerModal
   - Modal overlay with:
     - Video player (HTML5 <video> element with controls)
     - Title, description, author, timestamp
     - Like button (heart icon, toggles, shows count)
     - Comment section below:
       - List of comments (author avatar, name, text, timestamp)
       - "Add a comment" input at bottom
     - Close button (X)

4. Dashboard Integration:
   - Add an "Improvement Feed" card to the dashboard showing the 3 most recent videos as small thumbnails
   - Quick link to full feed page
   - If the employee hasn't recorded a video in 7+ days, show a gentle nudge: "Got a quick improvement to share? Record a 60-second video!"

TECHNICAL:
- Video recording: use MediaRecorder API with video/webm or video/mp4 codec
- If MediaRecorder outputs webm, that's fine — most browsers can play it
- Compress video before upload if possible (use canvas for thumbnails)
- Generate a thumbnail from the first frame of the video (draw video frame to canvas, export as JPEG)
- Upload flow: get presigned URL → PUT to URL → POST /api/videos with metadata
- For local storage fallback: POST multipart to /api/videos/upload → POST /api/videos with the returned path
- Loading skeletons while feed loads
- Infinite scroll or "Load More" button for the feed
- TypeScript strict: no `any` types
