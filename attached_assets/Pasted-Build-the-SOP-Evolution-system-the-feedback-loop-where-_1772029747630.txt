Build the SOP Evolution system — the feedback loop where employee submissions and AI insights trigger SOP revision proposals for owner review.

BACKEND:

1. DATABASE — Create migration:

TABLE: sop_revision_proposals
- id: uuid, primary key, default gen_random_uuid()
- store_id: text, not null, references stores(id)
- sop_template_id: uuid, not null, references sop_templates(id)
- source_type: text, not null (enum-like: 'what_bugged_you', 'sop_insight', 'ai_suggestion', 'manual')
- source_ids: jsonb, default '[]' (array of IDs: debrief IDs, insight IDs, etc.)
- proposal_type: text, not null (enum-like: 'add_step', 'remove_step', 'modify_step', 'reorder_steps', 'update_description', 'split_step', 'general')
- title: text, not null (short summary: "Split step 4 into two steps")
- description: text, not null (detailed explanation of the proposed change)
- ai_rationale: text, nullable (why the AI is suggesting this)
- proposed_changes: jsonb, nullable (structured change description if applicable)
- status: text, not null, default 'pending' (enum-like: 'pending', 'approved', 'rejected', 'implemented')
- reviewed_by: text, nullable (Clerk user ID)
- reviewed_at: timestamp with timezone, nullable
- review_notes: text, nullable
- created_at: timestamp with timezone, default now()

INDEX: sop_revision_proposals(store_id, status, created_at DESC)
INDEX: sop_revision_proposals(sop_template_id, status)

2. Create `server/services/sopEvolution.ts`:

FUNCTION: generateRevisionProposals(storeId: string): Promise<void>

Runs periodically (weekly, alongside SOP Intelligence analysis):

a. Gather "What Bugged You?" submissions from daily_debriefs that relate to SOPs:
   - Check what_bugged_you_category for 'process' submissions
   - Use semantic search to match debrief text against SOP content
   - Group submissions by matched SOP

b. Gather SOP insights (from sop_insights table) with severity 'warning' or 'action_needed'

c. For each SOP with multiple signals (2+ debrief mentions OR 1+ action_needed insight):
   - Call Claude to generate revision proposals:

Claude system prompt:
"""
You are MAinager's SOP Evolution engine. Given employee feedback and analytics data about a specific procedure, propose concrete revisions.

Rules:
- Each proposal must be specific enough that the owner can approve or reject it quickly
- Reference the actual employee feedback and data points
- Be constructive — you're improving procedures, not criticizing them
- Propose the SMALLEST change that would address the issue
- Never propose removing safety or compliance steps

Return JSON:
{
  "proposals": [
    {
      "proposal_type": "add_step" | "remove_step" | "modify_step" | "reorder_steps" | "update_description" | "split_step" | "general",
      "title": "Short description (max 10 words)",
      "description": "Detailed explanation of what to change and why (2-4 sentences)",
      "ai_rationale": "Data-backed reasoning: which employee feedback + which analytics support this change",
      "proposed_changes": { ...any structured detail... }
    }
  ]
}
"""

   User message: SOP title + steps + employee feedback quotes + analytics data

d. Create sop_revision_proposals records for each proposal
e. Notify the Owner via notification service: "MAinager has [X] SOP improvement suggestions for you"

3. API ENDPOINTS:

GET /api/sops/revisions — List revision proposals
- Auth: Owner/Admin
- Query params: status?, sop_template_id?
- Returns proposals with source context (the debrief quotes, insight data)

PUT /api/sops/revisions/:id — Review a proposal
- Auth: Owner only
- Zod body: { status: 'approved' | 'rejected', review_notes? }
- If approved: guide the owner to the SOP editor with the proposed changes pre-filled (or create a draft)

GET /api/sops/revisions/stats — Count of pending proposals (for dashboard badge)
- Auth: Owner/Admin

FRONTEND:

4. SOPRevisionProposalsPage (`/sops/revisions`)
   
   Layout:
   - List of pending proposals, grouped by SOP
   - Each proposal card:
     - SOP title reference
     - Proposal type badge (Add Step, Modify Step, Split Step, etc.)
     - Title and description
     - "Why this is suggested" expandable section showing:
       - Employee feedback quotes (anonymized): "An employee said: '[what bugged you text]'"
       - Analytics data point: "Step 4 is 3x slower than average"
     - Action buttons: "Approve & Edit SOP" / "Reject" / "Later"
   - Approved proposals show a link: "Edit SOP" → opens SOP builder with relevant section highlighted
   - Stats at top: X pending proposals across Y SOPs

5. SOP Detail Page Enhancement:
   - Add a badge on the SOP detail page if there are pending revision proposals
   - "Suggested Improvements ([count])" link → opens proposals for this SOP

6. Owner Dashboard Enhancement:
   - "SOP Improvement Suggestions" card with pending count
   - Click → SOPRevisionProposalsPage

DESIGN:
- This should feel like having a smart operations consultant reviewing your procedures weekly
- Proposals are suggestions, never demands — the owner always has final say
- Employee feedback is anonymized (never show who complained about what)
- Approval flow is fast — one tap to approve, owner then edits the SOP
