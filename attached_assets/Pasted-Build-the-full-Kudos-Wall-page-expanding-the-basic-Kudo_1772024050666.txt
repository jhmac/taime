Build the full Kudos Wall page — expanding the basic Kudos widget from Phase 1 into a dedicated celebration space.

Note: The kudos database table and basic API (POST /api/kudos, GET /api/kudos) were built in Phase 1 (Prompt 1.8/1.9). This prompt expands the frontend and adds a few backend enhancements.

BACKEND ADDITIONS:

1. Enhance GET /api/kudos:
   - Add query params: to_employee_id?, from_employee_id?, date_from?, date_to?
   - Include sender and receiver names/avatars in the response (join with Clerk user data or member directory)
   - Add: GET /api/kudos/stats — Kudos statistics
     - Per employee: total received, total given, this week received, this week given
     - Store totals: kudos this week, kudos this month, most recognized employee this month
     - Auth: Employee sees their own stats, Manager/Owner sees all

FRONTEND — Create `client/src/features/kudos/`:

1. KudosWallPage (`/kudos`)
   
   Design: feels like a social media feed of good vibes. Warm, celebratory, visually rich.
   
   Layout:
   - Header: "Kudos Wall ✨" with a warm gradient background
   - Stats bar: "[X] kudos this week • [Y] this month • Most recognized: [Name]"
   - "Give a Kudo" button (prominent, top right)
   
   Feed:
   - Scrollable feed of kudos cards
   - Each card:
     - Sender avatar + name → "gave a kudo to" → Receiver avatar + name
     - Message text (the kudo content)
     - Timestamp (relative)
     - Warm visual design: soft card with subtle border glow or heart accent
   - Filter options: All | Given by Me | Received by Me | This Week | This Month
   - Infinite scroll or "Load More"

2. GiveKudoDialog (enhanced from Phase 1):
   - Step 1: Who? — Employee avatar grid. Tap to select. Can search by name.
   - Step 2: What? — 
     - Quick-pick message templates: "Great teamwork!", "You crushed it today!", "Thanks for covering!", "Amazing customer service!", "Love the improvement!"
     - OR free text input (max 280 chars)
   - Step 3: Send
     - Preview: "You → [Name]: [message]"
     - "Send Kudo 💛" button
     - Celebration animation on send (confetti, hearts, something warm)
   - Success: "Kudo sent! [Name] will see it on the Kudos Wall."

3. Dashboard integration (enhance existing):
   - Employee dashboard: "You've received [X] kudos this month!" with latest kudo preview
   - Manager dashboard: "Team gave [X] kudos this week" — health indicator for team relatedness
   - Morning Huddle: kudos from last 24 hours already integrated from Phase 1

DESIGN:
- The wall should make people WANT to give kudos
- Every kudo feels like a little celebration
- No pressure to participate — purely positive space
- Mobile-first: easy to give kudos in 3 taps on a phone between customers
