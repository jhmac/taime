Build the SOP Template Builder UI — this is where store owners create and edit SOPs.

LOCATION: Create components in `client/src/features/sops/` (or follow existing feature folder pattern)

PAGES/COMPONENTS:

1. SOPLibraryPage (`/sops`)
   - Grid/list view of all SOP templates for the store
   - Filter by category (tabs or dropdown)
   - Search bar (searches title and description)
   - Each card shows: title, category badge, step count, estimated duration, last executed date
   - "Create New SOP" button (visible to Owner/Admin only)
   - Click a card → goes to SOP detail page

2. SOPBuilderPage (`/sops/new` and `/sops/:id/edit`)
   - Form with:
     - Title (required)
     - Description (textarea)
     - Category (select dropdown with the category options)
     - Estimated duration (number input, minutes)
     - Role assignments (multi-select checkboxes: Associate, Manager, Owner)
     - Training notes (textarea, collapsible section labeled "Training Mode Context — Why We Do This")
   - Step builder:
     - Sortable list of steps (drag-and-drop reordering using @dnd-kit/sortable or similar)
     - Each step has: title (required), description, step type (dropdown: Action, Verification, Photo, Decision, Timer), is_checkpoint toggle
     - Conditional fields based on step_type:
       - Timer: duration input (seconds)
       - Decision: add option labels and target step numbers
     - Training detail (collapsible per step)
     - "Add Step" button at the bottom
     - Delete step button (with confirmation)
   - Save button → POST /api/sops/templates
   - If editing: loads existing template, Save creates a new version (PUT endpoint)

3. SOPDetailPage (`/sops/:id`)
   - Shows the full SOP with all steps in order
   - Execution stats: times completed, avg duration, last executed
   - "Start Execution" button for employees
   - "Edit" button for Owner/Admin
   - Version history sidebar (list previous versions with dates)

DESIGN:
- Use shadcn/ui components: Card, Badge, Button, Input, Textarea, Select, Switch, Dialog
- Mobile-first responsive layout
- Category badges use distinct colors per category
- Step type icons: Action (✓), Verification (👁), Photo (📷), Decision (🔀), Timer (⏱)
- Follow the Emotional Design System: warm language, encouraging labels

ROUTING: Add these routes to the existing React Router setup.

TECHNICAL:
- Use React Query (or whatever data fetching pattern exists in the codebase) for API calls
- Zod validation on the form before submitting
- Loading states and error handling for all API calls
- TypeScript strict: define interfaces for all SOP types

DO NOT build the execution UI yet — that's the next prompt.
