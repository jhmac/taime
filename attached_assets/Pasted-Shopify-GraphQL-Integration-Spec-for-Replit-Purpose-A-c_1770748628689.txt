4. OAuth 2.0 Authentication Flow

This is the most critical piece. It connects a Shopify store to your app.

4.1 Callback URL Path Must Match Exactly

Your callback URL path must be IDENTICAL in three places:

Your Express route definition
The redirect_urls array in shopify.app.toml
The "Allowed redirection URL(s)" field in Shopify Partner Dashboard
A mismatch here is the #1 cause of OAuth failures.

4.2 Flow Overview

User clicks "Connect Store"
  → Your app stores user ID + CSRF state in session, then redirects to Shopify
  → User approves permissions on Shopify
  → Shopify redirects back to your callback URL with a code + HMAC
  → Your app verifies CSRF state and HMAC signature
  → Your app exchanges the code for a permanent access token (form-encoded POST)
  → Token is encrypted with AES-256-GCM before storing in database
  → Shop is linked to authenticated user via user_shops table
  → Webhooks are auto-registered for product sync
  → User is redirected to success page
Token encryption: The OAuth callback encrypts tokens with encryptToken() before storing them in the database. The getShopifyCredentials function passes tokens through decryptToken(), which decrypts enc:-prefixed tokens and returns legacy plaintext tokens as-is (backward compatibility for tokens stored before encryption was added).
4.3 Idempotency Guard (Prevents "code already used" errors)

Browsers sometimes send duplicate requests to the callback URL. Without this guard, the second request fails with "authorization code was already used".

// server/routes.ts - Place at the top of your routes file
import crypto from 'crypto';
const processedAuthCodes = new Map<string, { timestamp: number; status: 'processing' | 'success' | 'failed' }>();
const CODE_TTL_MS = 60000;
setInterval(() => {
  const now = Date.now();
  for (const [code, data] of processedAuthCodes) {
    if (now - data.timestamp > CODE_TTL_MS) {
      processedAuthCodes.delete(code);
    }
  }
}, 30000);
4.4 Step 1: Initiate OAuth (GET /api/shopify/auth)

This is the exact production code running in routes.ts:

app.get("/api/shopify/auth", async (req, res) => {
  try {
    const { shop } = req.query;
    if (!shop || typeof shop !== 'string') {
      return res.status(400).json({ error: 'Shop domain required' });
    }
    const shopDomain = shop.toLowerCase().trim();
    if (!shopDomain.includes('.myshopify.com')) {
      return res.status(400).json({ error: 'Invalid Shopify domain' });
    }
    // NOTE: Production uses BRANDIMAGE_* prefix with SHOPIFY_* fallback
    // Adjust the prefix to match your app name
    const apiKey = process.env.BRANDIMAGE_SHOPIFY_API_KEY || process.env.SHOPIFY_API_KEY;
    const scopes = process.env.SHOPIFY_SCOPES || 'write_products,read_products';
    const host = req.get('host') || '';
    const protocol = host.includes('localhost') ? 'http' : 'https';
    const redirectUri = `${protocol}://${host}/api/shopify/auth/callback`;
    const state = crypto.randomBytes(16).toString('hex');
    (req.session as any).oauthState = state;
    // CRITICAL: Store userId in session BEFORE redirect
    // The user context is lost during Shopify's redirect - this preserves it
    // Production uses Clerk auth (req.clerkUserId). Adapt to your auth system:
    const userId = (req as any).clerkUserId; // or req.user?.id for passport, etc.
    if (userId) {
      (req.session as any).oauthUserId = userId;
    }
    // Save session before redirecting (ensures state is persisted)
    await new Promise<void>((resolve, reject) => {
      req.session.save((err) => err ? reject(err) : resolve());
    });
    const authUrl = new URL(`https://${shopDomain}/admin/oauth/authorize`);
    authUrl.searchParams.set('client_id', apiKey || '');
    authUrl.searchParams.set('scope', scopes);
    authUrl.searchParams.set('redirect_uri', redirectUri);
    authUrl.searchParams.set('state', state);
    res.redirect(authUrl.toString());
  } catch (error) {
    console.error('[Shopify OAuth] Error:', error);
    res.status(500).json({ error: 'OAuth initialization failed' });
  }
});
4.5 Step 2: Handle Callback (GET /api/shopify/auth/callback)

This is the exact production callback. Key points:

Uses application/x-www-form-urlencoded for the token exchange (most reliable)
Includes idempotency guard for duplicate requests
Links the shop to the authenticated user
Auto-registers webhooks after successful connection
app.get("/api/shopify/auth/callback", async (req, res) => {
  try {
    const { code, hmac, shop, state } = req.query;
    // IDEMPOTENCY CHECK: Prevent double-processing of authorization codes
    if (code && typeof code === 'string') {
      const existingCode = processedAuthCodes.get(code);
      if (existingCode) {
        if (existingCode.status === 'success') {
          return res.redirect(`/settings?success=connected&shop=${encodeURIComponent(String(shop))}`);
        }
        if (existingCode.status === 'processing') {
          for (let i = 0; i < 10; i++) {
            await new Promise(resolve => setTimeout(resolve, 500));
            const updated = processedAuthCodes.get(code);
            if (updated?.status === 'success') {
              return res.redirect(`/settings?success=connected&shop=${encodeURIComponent(String(shop))}`);
            }
            if (updated?.status === 'failed') break;
          }
        }
      }
      processedAuthCodes.set(code, { timestamp: Date.now(), status: 'processing' });
    }
    // Verify CSRF state parameter
    if (state !== (req.session as any).oauthState) {
      console.error('[Shopify OAuth] State mismatch - session lost during redirect');
      if (code && typeof code === 'string') {
        processedAuthCodes.set(code, { timestamp: Date.now(), status: 'failed' });
      }
      return res.redirect(`/settings?error=${encodeURIComponent('Session expired. Please try again.')}`);
    }
    if (!shop || !code || typeof shop !== 'string' || typeof code !== 'string') {
      return res.redirect(`/settings?error=${encodeURIComponent('Missing OAuth parameters')}`);
    }
    const shopDomain = shop.toLowerCase().trim();
    const apiKey = process.env.BRANDIMAGE_SHOPIFY_API_KEY || process.env.SHOPIFY_API_KEY;
    const apiSecret = process.env.BRANDIMAGE_SHOPIFY_API_SECRET || process.env.SHOPIFY_API_SECRET;
    // Verify HMAC signature from Shopify (timing-safe comparison)
    if (hmac && apiSecret) {
      const queryParams = { ...req.query };
      delete queryParams.hmac;
      // IMPORTANT: Auth middleware (e.g., Clerk) may inject extra query params
      // These MUST be removed or HMAC verification will fail
      delete queryParams.__clerk_handshake; // Clerk-specific; remove for your auth system
      const message = Object.keys(queryParams)
        .sort()
        .map(key => `${key}=${queryParams[key]}`)
        .join('&');
      const hash = crypto.createHmac('sha256', apiSecret)
        .update(message)
        .digest('hex');
      // SECURITY: Use timing-safe comparison to prevent timing attacks
      const hashBuffer = Buffer.from(hash, 'hex');
      const hmacBuffer = Buffer.from(String(hmac), 'hex');
      if (hashBuffer.length !== hmacBuffer.length || !crypto.timingSafeEqual(hashBuffer, hmacBuffer)) {
        console.error('[Shopify OAuth] HMAC verification failed');
        return res.redirect(`/settings?error=${encodeURIComponent('Security verification failed')}`);
      }
    }
    // Exchange authorization code for permanent access token
    // IMPORTANT: Use application/x-www-form-urlencoded (most reliable with Shopify)
    const tokenBody = new URLSearchParams({
      client_id: apiKey || '',
      client_secret: apiSecret || '',
      code: code,
    });
    const tokenResponse = await fetch(`https://${shopDomain}/admin/oauth/access_token`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
      body: tokenBody.toString(),
    });
    if (!tokenResponse.ok) {
      const errorBody = await tokenResponse.text();
      throw new Error(`Token exchange failed (${tokenResponse.status}): ${errorBody}`);
    }
    const tokenData = await tokenResponse.json();
    const { access_token: accessToken } = tokenData;
    if (!accessToken) {
      throw new Error('No access token received from Shopify');
    }
    // SECURITY: Encrypt the access token before storing in the database
    const { encryptToken } = await import('./utils/tokenEncryption');
    const encryptedToken = encryptToken(accessToken);
    // Store shop connection in database with encrypted token
    const existing = await db.select()
      .from(shops)
      .where(eq(shops.shopDomain, shopDomain))
      .limit(1);
    if (existing.length > 0) {
      await db.update(shops)
        .set({
          accessToken: encryptedToken,
          isActive: true,
          updatedAt: new Date(),
        })
        .where(eq(shops.shopDomain, shopDomain));
    } else {
      await db.insert(shops).values({
        shopDomain,
        accessToken: encryptedToken,
        isActive: true,
      });
    }
    // Link shop to authenticated user
    const userId = (req.session as any).oauthUserId;
    if (userId) {
      const existingLink = await db.select()
        .from(userShops)
        .where(and(eq(userShops.userId, userId), eq(userShops.shopDomain, shopDomain)))
        .limit(1);
      if (!existingLink || existingLink.length === 0) {
        await db.insert(userShops).values({ userId, shopDomain });
      }
      delete (req.session as any).oauthUserId;
    }
    // Clean up session
    delete (req.session as any).oauthState;
    await new Promise<void>((resolve, reject) => {
      req.session.save((err) => err ? reject(err) : resolve());
    });
    // Mark code as successfully processed
    if (code) {
      processedAuthCodes.set(code, { timestamp: Date.now(), status: 'success' });
    }
    res.redirect(`/settings?success=connected&shop=${encodeURIComponent(shopDomain)}`);
  } catch (error) {
    const { code } = req.query;
    if (code && typeof code === 'string') {
      processedAuthCodes.set(code, { timestamp: Date.now(), status: 'failed' });
    }
    console.error('[Shopify OAuth] Callback error:', error);
    if (!res.headersSent) {
      res.redirect(`/settings?error=connection_failed`);
    }
  }
});
4.6 Auto-Register Webhooks After OAuth

After a successful OAuth connection, register webhooks immediately so product changes sync automatically:

// Place this right after the successful OAuth redirect, inside the callback handler
// Production creates the ShopifyService with the new credentials and uses
// REPLIT_DOMAINS for the base URL (with req.get('host') as fallback)
try {
  const { ShopifyService } = await import('./services/shopifyService');
  const shopifyService = new ShopifyService(shopDomain, accessToken);
  // REPLIT_DOMAINS is auto-set by Replit and contains the app's public hostname
  const baseUrl = process.env.REPLIT_DOMAINS
    ? `https://${process.env.REPLIT_DOMAINS.split(',')[0]}`
    : `https://${req.get('host')}`;
  const callbackUrl = `${baseUrl}/api/webhook`;
  const webhookTopics = ['products/create', 'products/update', 'products/delete'];
  for (const topic of webhookTopics) {
    try {
      const result = await shopifyService.registerWebhook(callbackUrl, topic);
      if (result?.webhookSubscription?.id) {
        console.log(`Registered webhook: ${topic}`);
      } else if (result?.userErrors?.some((e: any) => e.message?.includes('already exists'))) {
        console.log(`Webhook ${topic} already registered`);
      } else {
        console.warn(`Failed to register ${topic}:`, result?.userErrors);
      }
    } catch (webhookError) {
      console.warn(`Error registering ${topic}:`, webhookError);
    }
  }
} catch (webhookSetupError) {
  console.error('Webhook setup failed (non-blocking):', webhookSetupError);
  // Don't fail the entire OAuth flow if webhooks fail
}

5. Access Token Security (AES-256-GCM Encryption)

This module encrypts Shopify access tokens at rest. The decryptToken function handles both encrypted tokens (prefixed with enc:) and legacy plaintext tokens, enabling gradual migration.

// server/utils/tokenEncryption.ts
import crypto from "crypto";
const ALGORITHM = "aes-256-gcm";
const IV_LENGTH = 16;
const AUTH_TAG_LENGTH = 16;
function getEncryptionKey(): Buffer {
  const keySource =
    process.env.TOKEN_ENCRYPTION_KEY ||
    process.env.SESSION_SECRET;
  if (!keySource) {
    console.error("[TokenEncryption] SECURITY WARNING: No TOKEN_ENCRYPTION_KEY or SESSION_SECRET found!");
    throw new Error("Encryption key not configured. Set TOKEN_ENCRYPTION_KEY or SESSION_SECRET environment variable.");
  }
  return crypto.scryptSync(keySource, "shopify-token-salt", 32);
}
export function encryptToken(token: string): string {
  const key = getEncryptionKey();
  const iv = crypto.randomBytes(IV_LENGTH);
  const cipher = crypto.createCipheriv(ALGORITHM, key, iv);
  let encrypted = cipher.update(token, "utf8", "hex");
  encrypted += cipher.final("hex");
  const authTag = cipher.getAuthTag();
  const combined = Buffer.concat([iv, authTag, Buffer.from(encrypted, "hex")]);
  return "enc:" + combined.toString("base64");
}
export function decryptToken(encryptedToken: string): string {
  if (!encryptedToken.startsWith("enc:")) {
    console.log("[TokenEncryption] Token not encrypted, returning as-is (legacy token)");
    return encryptedToken;
  }
  try {
    const key = getEncryptionKey();
    const combined = Buffer.from(encryptedToken.slice(4), "base64");
    if (combined.length < IV_LENGTH + AUTH_TAG_LENGTH) {
      console.error("[TokenEncryption] Invalid encrypted token format - too short");
      throw new Error("Invalid encrypted token format");
    }
    const iv = combined.subarray(0, IV_LENGTH);
    const authTag = combined.subarray(IV_LENGTH, IV_LENGTH + AUTH_TAG_LENGTH);
    const encrypted = combined.subarray(IV_LENGTH + AUTH_TAG_LENGTH);
    const decipher = crypto.createDecipheriv(ALGORITHM, key, iv);
    decipher.setAuthTag(authTag);
    let decrypted = decipher.update(encrypted.toString("hex"), "hex", "utf8");
    decrypted += decipher.final("utf8");
    return decrypted;
  } catch (error) {
    console.error("[TokenEncryption] Failed to decrypt token:", error instanceof Error ? error.message : "Unknown error");
    throw new Error("Token decryption failed - token may be corrupted or encrypted with different key");
  }
}
export function isTokenEncrypted(token: string): boolean {
  return token.startsWith("enc:");
}
export function maskToken(token: string, visibleChars: number = 4): string {
  if (!token) return "";
  const decrypted = isTokenEncrypted(token) ? "[encrypted]" : token;
  if (decrypted === "[encrypted]") return decrypted;
  if (decrypted.length <= visibleChars) {
    return "*".repeat(decrypted.length);
  }
  return decrypted.substring(0, visibleChars) + "*".repeat(decrypted.length - visibleChars);
}
Getting Decrypted Credentials for API Calls

This is the production pattern for retrieving credentials. It checks both the shops table and falls back to shopSessions, and handles both encrypted and plaintext tokens:

// server/routes.ts (exported function)
import { decryptToken } from './utils/tokenEncryption';
export async function getShopifyCredentials(
  shopDomain: string
): Promise<{ shopDomain: string; accessToken: string } | null> {
  try {
    const normalizedDomain = shopDomain.trim().toLowerCase();
    // Primary: shops table (where OAuth stores tokens)
    const shopResult = await db.select()
      .from(shops)
      .where(eq(shops.shopDomain, normalizedDomain))
      .limit(1);
    if (shopResult.length > 0 && shopResult[0].accessToken) {
      try {
        const decryptedToken = decryptToken(shopResult[0].accessToken);
        return {
          shopDomain: shopResult[0].shopDomain,
          accessToken: decryptedToken,
        };
      } catch (decryptError) {
        console.error(`Failed to decrypt token for ${normalizedDomain}:`, decryptError);
        return null;
      }
    }
    // Fallback: shopSessions table
    const sessionResult = await db.select()
      .from(shopSessions)
      .where(eq(shopSessions.shopDomain, normalizedDomain))
      .limit(1);
    if (sessionResult.length > 0 && sessionResult[0].accessToken) {
      try {
        const decryptedToken = decryptToken(sessionResult[0].accessToken);
        return {
          shopDomain: sessionResult[0].shopDomain,
          accessToken: decryptedToken,
        };
      } catch (decryptError) {
        console.error(`Failed to decrypt token for ${normalizedDomain}:`, decryptError);
        return null;
      }
    }
    console.error(`No token found for: ${normalizedDomain}`);
    return null;

        }

   *

   * - Uses X-Shopify-Access-Token header (NOT Authorization: Bearer)
   * - Endpoint: https://{shop}/admin/api/{version}/graphql.json
   * - Always check for data.errors even on 200 responses
   * - Shopify returns 200 with errors for invalid tokens
   */
  async makeGraphQLRequest(query: string, variables?: Record<string, any>): Promise<any> {
    const url = `https://${this.shopDomain}/admin/api/${SHOPIFY_API_VERSION}/graphql.json`;
    const response = await fetch(url, {
      method: 'POST',
      headers: {
        'X-Shopify-Access-Token': this.accessToken,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ query, variables }),
    });
    if (!response.ok) {
      const errorText = await response.text();
      throw new Error(`Shopify API error (${response.status}): ${errorText}`);
    }
    const data = await response.json();
    if (data.errors) {
      throw new Error(`Shopify GraphQL error: ${data.errors[0]?.message || 'Unknown error'}`);
    }
    return data;
  }

8.2 Upload Image from Buffer (3-Step Staged Upload)

Required when you have an image file/buffer (e.g., from AI generation or user upload).

private async uploadMediaFromBuffer(productId: string, imageBuffer: Buffer, altText?: string): Promise<any> {
  // STEP 1: Create a staged upload target
  const stagedUploadMutation = `
    mutation stagedUploadsCreate($input: [StagedUploadInput!]!) {
      stagedUploadsCreate(input: $input) {
        stagedTargets {
          url
          resourceUrl
          parameters {
            name
            value
          }
        }
        userErrors {
          field
          message
        }
      }
    }
  `;
  const stagedUploadVars = {
    input: [{
      resource: 'PRODUCT_IMAGE',
      filename: `ai-generated-${Date.now()}.jpg`,
      mimeType: 'image/jpeg',
      httpMethod: 'POST',
    }],
  };
  const stagedUploadResponse = await this.makeGraphQLRequest(stagedUploadMutation, stagedUploadVars);
  if (stagedUploadResponse.data?.stagedUploadsCreate?.userErrors?.length > 0) {
    const errors = stagedUploadResponse.data.stagedUploadsCreate.userErrors;
    throw new Error(`Failed to create staged upload: ${errors.map((e: any) => e.message).join(', ')}`);
  }
  const stagedTargets = stagedUploadResponse.data?.stagedUploadsCreate?.stagedTargets;
  if (!stagedTargets || stagedTargets.length === 0) {
    throw new Error('No staged upload targets returned from Shopify');
  }
  const { url: uploadUrl, resourceUrl, parameters } = stagedTargets[0];
  // STEP 2: Upload file to GCS via multipart form
  const boundary = `----WebKitFormBoundary${Math.random().toString(36).substr(2, 16)}`;
  const chunks: Buffer[] = [];
  // Add all Shopify-provided parameters FIRST (they include auth tokens for GCS)
  for (const param of parameters) {
    chunks.push(Buffer.from(`--${boundary}\r\n`));
    chunks.push(Buffer.from(`Content-Disposition: form-data; name="${param.name}"\r\n\r\n`));
    chunks.push(Buffer.from(`${param.value}\r\n`));
  }
  // Add the actual file LAST (required by GCS)
  chunks.push(Buffer.from(`--${boundary}\r\n`));
  chunks.push(Buffer.from(`Content-Disposition: form-data; name="file"; filename="image.jpg"\r\n`));
  chunks.push(Buffer.from(`Content-Type: image/jpeg\r\n\r\n`));
  chunks.push(imageBuffer);
  chunks.push(Buffer.from(`\r\n--${boundary}--\r\n`));
  const body = Buffer.concat(chunks);
  const uploadResponse = await fetch(uploadUrl, {
    method: 'POST',
    headers: {
      'Content-Type': `multipart/form-data; boundary=${boundary}`,
      'Content-Length': body.length.toString(),
    },
    body: body,
  });
  if (!uploadResponse.ok) {
    const errorText = await uploadResponse.text();
    throw new Error(`Failed to upload file to staging: ${uploadResponse.status} - ${errorText}`);
  }
  // STEP 3: Attach the uploaded file to the product as media
  const createMediaMutation = `
    mutation productCreateMedia($media: [CreateMediaInput!]!, $productId: ID!) {
      productCreateMedia(media: $media, productId: $productId) {
        media {
          ... on MediaImage {
            id
            image {
              url
              altText
            }
          }
        }
        userErrors {
          field
          message
        }
      }
    }
  `;
  const createMediaVars = {
    productId,
    media: [{
      originalSource: resourceUrl,  // Use the resourceUrl from step 1
      alt: altText || '',
      mediaContentType: 'IMAGE',
    }],
  };
  const mediaResponse = await this.makeGraphQLRequest(createMediaMutation, createMediaVars);
  if (mediaResponse.data?.productCreateMedia?.userErrors?.length > 0) {
    const errors = mediaResponse.data.productCreateMedia.userErrors;
    throw new Error(`Failed to create media: ${errors.map((e: any) => e.message).join(', ')}`);
  }
  return mediaResponse.data?.productCreateMedia?.media;
}

8.4 Reorder Product Images

CRITICAL: This is an async operation. Shopify returns a job ID that must be polled.

async updateImageOrder(
  productId: string,
  imageOrder: Array<{ id: string; position: number }>
): Promise<void> {
  const graphqlQuery = `
    mutation productReorderMedia($id: ID!, $moves: [MoveInput!]!) {
      productReorderMedia(id: $id, moves: $moves) {
        job {
          id
          done
        }
        mediaUserErrors {
          field
          message
          code
        }
      }
    }
  `;
  // IMPORTANT: Caller must provide valid MediaImage IDs (gid://shopify/MediaImage/...)
  // IMPORTANT: newPosition must be a String per Shopify API
  const moves = imageOrder.map(item => {
    if (!item.id.includes('MediaImage')) {
      console.warn(`WARNING: Expected MediaImage ID but got: ${item.id}`);
    }
    return {
      id: item.id,
      newPosition: String(item.position),
    };
  });
  const variables = {
    id: productId.startsWith('gid://') ? productId : `gid://shopify/Product/${productId}`,
    moves,
  };
  const response = await this.makeGraphQLRequest(graphqlQuery, variables);
  if (response.data?.productReorderMedia?.mediaUserErrors?.length > 0) {
    const errors = response.data.productReorderMedia.mediaUserErrors;
    throw new Error(`Failed to reorder images: ${errors.map((e: any) => e.message).join(', ')}`);
  }
  // Poll the job until it completes
  const jobId = response.data?.productReorderMedia?.job?.id;
  if (jobId) {
    await this.pollJobUntilDone(jobId, 15000);
  }
}
private async pollJobUntilDone(jobId: string, timeoutMs: number = 15000): Promise<void> {
  const jobQuery = `
    query getJob($id: ID!) {
      job(id: $id) {
        id
        done
      }
    }
  `;
  const startTime = Date.now();
  let delay = 500;
  while (Date.now() - startTime < timeoutMs) {
    const response = await this.makeGraphQLRequest(jobQuery, { id: jobId });
    const job = response.data?.job;
    if (job?.done) return;
    await new Promise(resolve => setTimeout(resolve, delay));
    delay = Math.min(delay * 1.5, 2000);
  }
  console.warn(`Job polling timed out: ${jobId}`);
}

Exact production code. Note: Shopify order date filters use space-separated syntax, NOT AND.

async getOrders(filters: {
  first?: number;
  createdAtMin?: string;
  createdAtMax?: string;
  maxPages?: number;
} = {}): Promise<any[]> {
  const { first = 50, createdAtMin, createdAtMax, maxPages = 10 } = filters;
  let queryFilter = '';
  const filterParts: string[] = [];
  if (createdAtMin) {
    const dateOnly = createdAtMin.split('T')[0];
    filterParts.push(`created_at:>=${dateOnly}`);
  }
  if (createdAtMax) {
    const dateOnly = createdAtMax.split('T')[0];
    filterParts.push(`created_at:<=${dateOnly}`);
  }
  if (filterParts.length > 0) {
    queryFilter = filterParts.join(' ');  // Space-separated, NOT AND
  }
  const graphqlQuery = `
    query GetOrders($first: Int!, $query: String, $after: String) {
      orders(first: $first, query: $query, sortKey: CREATED_AT, reverse: true, after: $after) {
        nodes {
          id
          name
          createdAt
          totalPriceSet {
            shopMoney {
              amount
              currencyCode
            }
          }
          subtotalPriceSet {
            shopMoney {
              amount
              currencyCode
            }
          }
          displayFinancialStatus
          displayFulfillmentStatus
          customer {
            id
            displayName
            email
          }
          lineItems(first: 10) {
            nodes {
              title
              quantity
              variant {
                price
                product {
                  id
                  title
                }
              }
            }
          }
        }
        pageInfo {
          hasNextPage
          endCursor
        }
      }
    }
  `;
  const allOrders: any[] = [];
  let cursor: string | null = null;
  let pageCount = 0;
  try {
    do {
      pageCount++;
      const variables: Record<string, any> = { first };
      if (queryFilter) variables.query = queryFilter;
      if (cursor) variables.after = cursor;
      const response = await this.makeGraphQLRequest(graphqlQuery, variables);
      const orders = response.data?.orders?.nodes || [];
      const pageInfo = response.data?.orders?.pageInfo;
      allOrders.push(...orders);
      if (pageInfo?.hasNextPage && pageInfo?.endCursor && pageCount < maxPages) {
        cursor = pageInfo.endCursor;
      } else {
        cursor = null;
      }
    } while (cursor);
    return allOrders;
  } catch (error) {
    console.error('Error fetching orders:', error);
    return allOrders; // Return what we have so far
  }

9.2 Sales Summary (Aggregated Analytics)

async getSalesSummary(options: { daysBack?: number } = {}): Promise<{
  totalOrders: number;
  totalRevenue: number;
  averageOrderValue: number;
  currency: string;
  periodStart: string;
  periodEnd: string;
  topProducts: Array<{ title: string; quantity: number; revenue: number }>;
}> {
  const { daysBack = 30 } = options;
  const now = new Date();
  const startDate = new Date(now.getTime() - daysBack * 24 * 60 * 60 * 1000);
  const createdAtMin = startDate.toISOString();
  const createdAtMax = now.toISOString();
  const orders = await this.getOrders({ first: 250, createdAtMin, createdAtMax });
  let totalRevenue = 0;
  let currency = 'USD';
  const productSales: Record<string, { title: string; quantity: number; revenue: number }> = {};
  for (const order of orders) {
    const orderTotal = parseFloat(order.totalPriceSet?.shopMoney?.amount || '0');
    totalRevenue += orderTotal;
    currency = order.totalPriceSet?.shopMoney?.currencyCode || 'USD';
    for (const lineItem of (order.lineItems?.nodes || [])) {
      const productTitle = lineItem.variant?.product?.title || lineItem.title || 'Unknown';
      const quantity = lineItem.quantity || 1;
      const price = parseFloat(lineItem.variant?.price || '0');
      const lineRevenue = price * quantity;
      if (!productSales[productTitle]) {
        productSales[productTitle] = { title: productTitle, quantity: 0, revenue: 0 };
      }
      productSales[productTitle].quantity += quantity;
      productSales[productTitle].revenue += lineRevenue;
    }
  }
  const topProducts = Object.values(productSales)
    .sort((a, b) => b.revenue - a.revenue)
    .slice(0, 10);
  const averageOrderValue = orders.length > 0 ? totalRevenue / orders.length : 0;
  return {
    totalOrders: orders.length,
    totalRevenue: Math.round(totalRevenue * 100) / 100,
    averageOrderValue: Math.round(averageOrderValue * 100) / 100,
    currency,
    periodStart: createdAtMin,
    periodEnd: createdAtMax,
    topProducts,
  };
}

13. Cursor-Based Pagination

Shopify uses cursor-based pagination (NOT offset-based). Production pattern:

async fetchAllPaginated<T>(
  queryTemplate: string,
  resourceName: string,
  pageSize: number = 50,
  maxPages: number = 20,
): Promise<T[]> {
  const allItems: T[] = [];
  let cursor: string | null = null;
  let page = 0;
  do {
    page++;
    const variables: Record<string, any> = { first: Math.min(pageSize, 250) };
    if (cursor) variables.after = cursor;
    const response = await this.makeGraphQLRequest(queryTemplate, variables);
    const items = response.data?.[resourceName]?.nodes || [];
    const pageInfo = response.data?.[resourceName]?.pageInfo;
    allItems.push(...items);
    if (pageInfo?.hasNextPage && pageInfo?.endCursor && page < maxPages) {
      cursor = pageInfo.endCursor;
    } else {
      cursor = null;
    }
  } while (cursor);
  return allItems;
}
Key rules:

Maximum first value is 250
Always use after cursor from pageInfo.endCursor
Check pageInfo.hasNextPage to know when to stop

14.2 Webhook Receiver Endpoint

CRITICAL: rawBody Middleware Requirement

Shopify webhook HMAC verification computes a hash of the raw request body (the exact bytes). If you parse JSON first (express.json()), you lose access to the original bytes, and HMAC verification will ALWAYS fail. You MUST capture the raw body BEFORE any JSON parsing.
// Place this BEFORE express.json() or scope to webhook routes only:
app.use('/api/webhook', (req, res, next) => {
  let data = '';
  req.on('data', chunk => { data += chunk; });
  req.on('end', () => {
    (req as any).rawBody = data;
    try {
      req.body = JSON.parse(data);
    } catch {
      req.body = {};
    }
    next();
  });
});
app.post('/api/webhook', async (req, res) => {
  const hmac = req.headers['x-shopify-hmac-sha256'] as string;
  const topic = req.headers['x-shopify-topic'] as string;
  const shopDomain = req.headers['x-shopify-shop-domain'] as string;
  if (!verifyWebhookHmac((req as any).rawBody, hmac)) {

14.3 Webhook HMAC Verification

Exact production code from gdprWebhooks.ts. Note: webhook HMAC uses base64 encoding (NOT hex like OAuth).

function verifyWebhookHmac(rawBody: string, hmac: string): boolean {
  const SHOPIFY_API_SECRET = (process.env.BRANDIMAGE_SHOPIFY_API_SECRET || process.env.SHOPIFY_API_SECRET)!;
  if (!hmac || !rawBody) return false;
  const hash = crypto
    .createHmac('sha256', SHOPIFY_API_SECRET)
    .update(rawBody, 'utf8')
    .digest('base64');
  const hashBuffer = Buffer.from(hash, 'base64');
  const hmacBuffer = Buffer.from(hmac, 'base64');
  if (hashBuffer.length !== hmacBuffer.length) return false;
  return crypto.timingSafeEqual(hashBuffer, hmacBuffer);
}

20. Common Pitfalls & Solutions

Pitfall 1: "Code already used" errors

Problem: Browser sends duplicate requests to the callback URL. Solution: Idempotency guard with a Map tracking processed codes (see Section 4.3).

Pitfall 2: Session lost during OAuth redirect

Problem: Session cookie not preserved when Shopify redirects back. Solution: Set sameSite: 'lax' on your session cookie. Store user ID in session before redirect.

Pitfall 3: HMAC verification fails due to extra query parameters

Problem: Auth middleware (e.g., Clerk) adds extra query parameters that break HMAC calculation. Solution: Delete any non-Shopify query parameters before computing the HMAC. Example: delete queryParams.__clerk_handshake;

Pitfall 4: Product ID format mismatch

Problem: REST API uses numeric IDs (12345), GraphQL uses GIDs (gid://shopify/Product/12345). Solution: Always convert: const gid = id.startsWith('gid://') ? id : \gid://shopify/Product/${id}``;

Pitfall 5: Image reorder uses wrong ID type

Problem: productReorderMedia requires MediaImage IDs, not ProductImage IDs. Solution: Use media(first: 20) query (not images) and use the MediaImage id field.

Pitfall 6: Staged upload file must come last

Problem: File upload to GCS fails with 403 or corrupt data. Solution: In the multipart form, add all Shopify-provided parameters FIRST, then the file LAST.

Pitfall 7: GraphQL 200 OK but with errors

Problem: Shopify returns HTTP 200 even when the query fails. Solution: Always check response.data.errors in addition to HTTP status.

Pitfall 8: newPosition must be a String

Problem: productReorderMedia mutation fails silently. Solution: Cast position to String: newPosition: String(position).

Pitfall 9: productsCount requires limit parameter

Problem: productsCount field on collections returns error. Solution: Use productsCount(limit: null) { count }.

Pitfall 10: Token exchange Content-Type

Problem: Token exchange returns 400 or empty response. Solution: Use application/x-www-form-urlencoded with URLSearchParams for the token exchange body (production-verified).

Pitfall 11: Order date filter syntax

Problem: Order queries return wrong results or errors when filtering by date. Solution: Use space-separated filters (NOT AND): created_at:>=2024-01-01 created_at:<=2024-12-31. Also convert ISO dates to YYYY-MM-DD format for Shopify.

Pitfall 12: Webhook HMAC uses base64, OAuth HMAC uses hex

Problem: HMAC verification fails for webhooks or OAuth. Solution: OAuth callback HMAC is hex-encoded; webhook HMAC (via X-Shopify-Hmac-Sha256 header) is base64-encoded. These are different encodings.

22. Delegate Access Tokens (Principle of Least Privilege)

Shopify's delegate access tokens let you create subsystem-specific tokens with only the scopes each subsystem needs, instead of using the full parent token everywhere. This follows the principle of least privilege - if a delegate token is compromised, the blast radius is limited to only the scopes it was granted.

22.2 Creating Delegate Tokens via GraphQL

The delegateAccessTokenCreate mutation creates a scoped token from the parent token. Returns null on failure (the graceful fallback to parent token happens in a separate helper method - see Section 22.4):

// Inside DelegateTokenService class
static async createDelegateToken(
  shopDomain: string,
  subsystem: keyof typeof SUBSYSTEM_SCOPE_MAP
): Promise<{ accessToken: string; scopes: string[]; expiresIn: number } | null> {
  const scopeConfig = SUBSYSTEM_SCOPE_MAP[subsystem];
  if (!scopeConfig) {
    console.error(`[DelegateTokenService] Unknown subsystem: ${subsystem}`);
    return null;
  }
  // Check cache first (with 60s buffer before expiry)
  const cacheKey = getCacheKey(shopDomain, subsystem);
  const cached = delegateTokenCache.get(cacheKey);
  if (cached && Date.now() < cached.expiresAt - 60000) {
    return {
      accessToken: cached.accessToken,
      scopes: cached.scopes,
      expiresIn: Math.floor((cached.expiresAt - Date.now()) / 1000),
    };
  }
  const credentials = await getShopifyCredentials(shopDomain);
  if (!credentials) {
    console.error(`[DelegateTokenService] No credentials found for ${shopDomain}`);
    return null;
  }
  const expiresIn = scopeConfig.expiresIn || 3600;
  try {
    const mutation = `
      mutation delegateAccessTokenCreate($input: DelegateAccessTokenInput!) {
        delegateAccessTokenCreate(input: $input) {
          delegateAccessToken {
            accessToken
            accessScopes
            createdAt
          }
          shop {
            id
          }
          userErrors {
            field
            message
          }
        }
      }
    `;
    const variables = {
      input: {
        delegateAccessScope: scopeConfig.scopes,
        expiresIn: expiresIn,
      },
    };
    const url = `https://${credentials.shopDomain}/admin/api/${SHOPIFY_API_VERSION}/graphql.json`;
    const response = await fetch(url, {
      method: 'POST',
      headers: {
        'X-Shopify-Access-Token': credentials.accessToken, // Parent token
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ query: mutation, variables }),
    });
    if (!response.ok) {
      const errorText = await response.text();
      console.error(`[DelegateTokenService] API error (${response.status}):`, errorText);
      return null;
    }
    const data = await response.json();
    if (data.errors) {
      console.error(`[DelegateTokenService] GraphQL errors:`, data.errors);
      return null;
    }
    const result = data.data?.delegateAccessTokenCreate;
    if (result?.userErrors?.length > 0) {
      console.error(`[DelegateTokenService] User errors:`, result.userErrors);
      return null;
    }
    const delegateToken = result?.delegateAccessToken;
    if (!delegateToken?.accessToken) {
      console.error(`[DelegateTokenService] No delegate token returned`);
      return null;
    }
    // Cache the token
    delegateTokenCache.set(cacheKey, {
      accessToken: delegateToken.accessToken,
      subsystem,
      scopes: scopeConfig.scopes,
      createdAt: Date.now(),
      expiresAt: Date.now() + (expiresIn * 1000),
    });
    return {
      accessToken: delegateToken.accessToken,
      scopes: scopeConfig.scopes,
      expiresIn,
    };
  } catch (error) {
    console.error(`[DelegateTokenService] Error creating delegate token:`, error);
    return null; // Caller decides whether to fall back to parent token
  }
}
22.3 Token Caching Strategy

Delegate tokens are cached in-memory with automatic cleanup every 5 minutes:

interface CachedDelegateToken {
  accessToken: string;
  subsystem: string;
  scopes: string[];
  createdAt: number;
  expiresAt: number;
}
const delegateTokenCache = new Map<string, CachedDelegateToken>();
const CACHE_CLEANUP_INTERVAL = 5 * 60 * 1000;
setInterval(() => {
  const now = Date.now();
  Array.from(delegateTokenCache.entries()).forEach(([key, token]) => {
    if (now >= token.expiresAt) {
      delegateTokenCache.delete(key);
    }
  });
}, CACHE_CLEANUP_INTERVAL);
function getCacheKey(shopDomain: string, subsystem: string): string {
  return `${shopDomain}:${subsystem}`;
}
22.4 Graceful Fallback Helper

Use getShopifyServiceWithDelegateToken to get a ShopifyService that automatically falls back to the parent token if delegate token creation fails:

static async getShopifyServiceWithDelegateToken(
  shopDomain: string,
  subsystem: keyof typeof SUBSYSTEM_SCOPE_MAP
): Promise<ShopifyService | null> {
  const delegateResult = await this.createDelegateToken(shopDomain, subsystem);
  if (!delegateResult) {
    // GRACEFUL FALLBACK: Use parent token if delegate creation fails
    console.warn(`[DelegateTokenService] Delegate token creation failed for ${subsystem}, falling back to parent token`);
    const credentials = await getShopifyCredentials(shopDomain);
    if (!credentials) return null;
    return new ShopifyService(credentials.shopDomain, credentials.accessToken);
  }
  return new ShopifyService(shopDomain, delegateResult.accessToken);
}

  const { shopDomain, subsystem } = req.body;
  const { DelegateTokenService } = await import('./services/delegateTokenService');
  const result = await DelegateTokenService.createDelegateToken(shopDomain, subsystem);
  if (!result) return res.status(500).json({ error: 'Failed to create delegate token' });
  res.json({
    subsystem,
    scopes: result.scopes,
    expiresIn: result.expiresIn,
    message: `Delegate token created for ${subsystem} with ${result.scopes.length} scopes`,
  });
});
23. Scope Revocation (Reducing Access Post-Install)

After initial OAuth, you can revoke scopes your app no longer needs using the appRevokeAccessScopes mutation. This is important for security hygiene - if you stop using a feature, revoke its scope.

23.1 Revoke Specific Scopes

static async revokeAccessScopes(
  shopDomain: string,
  scopesToRevoke: string[]
): Promise<{ revoked: string[]; errors: string[] }> {
  const credentials = await getShopifyCredentials(shopDomain);
  if (!credentials) {
    return { revoked: [], errors: ['No credentials found'] };
  }
  const mutation = `
    mutation appRevokeAccessScopes($scopes: [String!]!) {
      appRevokeAccessScopes(scopes: $scopes) {
        revoked {
          handle
          description
        }
        userErrors {
          field
          message
        }
      }
    }
  `;
  const url = `https://${credentials.shopDomain}/admin/api/2025-04/graphql.json`;
  const response = await fetch(url, {
    method: 'POST',
    headers: {
      'X-Shopify-Access-Token': credentials.accessToken,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      query: mutation,
      variables: { scopes: scopesToRevoke },
    }),
  });
  const data = await response.json();
  const result = data.data?.appRevokeAccessScopes;
  if (result?.userErrors?.length > 0) {
    return { revoked: [], errors: result.userErrors.map((e: any) => e.message) };
  }
  const revokedHandles = (result?.revoked || []).map((s: any) => s.handle);
  // Invalidate delegate token cache after scope change
  DelegateTokenService.invalidateCache(shopDomain);
  return { revoked: revokedHandles, errors: [] };
}
23.2 Query Currently Granted Scopes

static async getGrantedScopes(shopDomain: string): Promise<{
  scopes: Array<{ handle: string; description: string }>;
  errors: string[];
}> {
  const credentials = await getShopifyCredentials(shopDomain);
  if (!credentials) return { scopes: [], errors: ['No credentials found'] };
  const query = `{
    appInstallation {
      accessScopes {
        handle
        description
      }
    }
  }`;
  const url = `https://${credentials.shopDomain}/admin/api/2025-04/graphql.json`;
  const response = await fetch(url, {
    method: 'POST',
    headers: {
      'X-Shopify-Access-Token': credentials.accessToken,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ query }),
  });
  const data = await response.json();
  return {
    scopes: data.data?.appInstallation?.accessScopes || [],
    errors: data.errors?.map((e: any) => e.message) || [],
  };
}
23.3 Scope Management API Endpoints

// GET /api/shopify/scopes?shop=example.myshopify.com - View granted scopes
app.get("/api/shopify/scopes", isClerkAuthenticated, async (req, res) => {
  const shopDomain = req.query.shop as string;
  if (!shopDomain) {
    return res.status(400).json({ error: 'Shop domain required' });
  }
  const { DelegateTokenService } = await import('./services/delegateTokenService');
  const result = await DelegateTokenService.getGrantedScopes(shopDomain);
  if (result.errors.length > 0) {
    return res.status(500).json({ error: result.errors.join(', ') });
  }
  res.json({ scopes: result.scopes });
});
// POST /api/shopify/scopes/revoke - Revoke specific scopes
app.post("/api/shopify/scopes/revoke", isClerkAuthenticated, async (req, res) => {
  const revokeSchema = z.object({
    shopDomain: z.string().min(1),
    scopes: z.array(z.string().min(1)).min(1),
  });
  const parseResult = revokeSchema.safeParse(req.body);
  if (!parseResult.success) {
    return res.status(400).json({ error: parseResult.error.errors[0]?.message || 'Invalid request' });
  }
  const { shopDomain, scopes } = parseResult.data;
  const { DelegateTokenService } = await import('./services/delegateTokenService');
  const result = await DelegateTokenService.revokeAccessScopes(shopDomain, scopes);
  if (result.errors.length > 0) {
    return res.status(400).json({ error: result.errors.join(', '), revoked: result.revoked });
  }
  res.json({ revoked: result.revoked, message: `Successfully revoked ${result.revoked.length} scope(s)` });
});

Appendix B: Shopify GraphQL ID Format Reference

All Shopify resources use Global ID (GID) format in GraphQL:

Resource	GID Format
Product	gid://shopify/Product/12345
ProductVariant	gid://shopify/ProductVariant/12345
MediaImage	gid://shopify/MediaImage/12345
ProductImage	gid://shopify/ProductImage/12345
Collection	gid://shopify/Collection/12345
Order	gid://shopify/Order/12345
Customer	gid://shopify/Customer/12345
AppSubscription	gid://shopify/AppSubscription/12345
Job	gid://shopify/Job/12345
Always use GID format in GraphQL queries. Never use bare numeric IDs.

Appendix C: HMAC Encoding Reference

Two different HMAC encodings are used - this is a common source of bugs:

Context	Algorithm	Encoding	Source
OAuth callback	HMAC-SHA256	hex	Query param hmac
Webhooks	HMAC-SHA256	base64	Header X-Shopify-Hmac-Sha256
