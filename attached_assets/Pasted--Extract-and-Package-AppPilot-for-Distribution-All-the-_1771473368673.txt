# Extract and Package AppPilot for Distribution

All the latest features (ELON, Playwright crawler, continuous loop, ELON-REPORT, dashboard buttons) are working in this app’s `node_modules/apppilot/`. I need to extract it into a clean package that I can install in other Replit apps.

## Step 1: Copy patched source out of node_modules

```bash
rm -rf ~/workspace/apppilot-package
mkdir -p ~/workspace/apppilot-package
cp -r node_modules/apppilot/* ~/workspace/apppilot-package/
```

## Step 2: Clean up the package

In `~/workspace/apppilot-package/`:

1. **Remove debug code** — search all files in `src/` and `bin/` for:
- `console.log` statements that were added for debugging (keep intentional logging like cycle progress and error messages)
- Any `// TODO` or `// HACK` or `// TEMP` comments with temporary code
- Any hardcoded paths like `/home/runner/workspace` — replace with relative paths or `process.cwd()`
1. **Verify no app-specific data leaked in** — check that no ShopTok-specific code, API keys, store IDs, or customer data ended up in the package files. The package should be completely app-agnostic.
1. **Verify all new files exist:**
- `src/elon.js` — ELON constraint solver engine
- `src/site-crawler.js` — Playwright site crawler
- `bin/elon.js` — ELON CLI command
- `bin/crawl.js` — Crawl CLI command
- `bin/continuous.js` — Continuous loop CLI command
- `bin/apppilot.js` — Init CLI command (existing)
- `bin/heartbeat.js` — Heartbeat CLI command (existing)
- `templates/subagents/elon.md` — ELON subagent template
- `src/dashboard/index.html` — Dashboard with ELON, Crawl, Discovery buttons
1. **Make sure all template files are generic** — check `templates/` folder:
- `GOALS-template.md` should have placeholder text, not ShopTok content
- `SOUL-template.md` should have placeholder owner email
- `AGENTS-template.md` should have generic safe paths like `src/**`, `server/**`
- No ShopTok-specific references in any template file

## Step 5: Write a README

Create `~/workspace/apppilot-package/README.md`:

```markdown
# AppPilot

Autonomous AI agent that monitors, analyzes, and improves your app. Uses Claude to find issues, generate fixes, and apply them — with rollback safety.

## Install

npm install ./apppilot-0.3.0.tgz

## Setup

npx apppilot init
npx playwright install chromium

Edit GOALS.md, AGENTS.md, and SOUL.md for your app.

Set environment variables:
- ANTHROPIC_API_KEY (required)
- ANTHROPIC_BASE_URL (if using Replit AI proxy)

## Commands

| Command | What it does |
|---------|-------------|
| npx apppilot init | Set up AppPilot in your project |
| npx apppilot-heartbeat | Run one monitoring + fix cycle |
| npx apppilot-elon | Find and solve limiting factors with Playwright |
| npx apppilot-crawl | Crawl your site and report errors |
| npx apppilot-continuous | Keep fixing until all high/medium issues are resolved |

## Features

- **Ralph Loop** — specs go in, code changes come out, tests validate
- **ELON** — strategic constraint solver that finds what's most blocking your goals
- **Playwright Crawler** — crawls your live site like a real user to find errors
- **Auto-approval** — safe changes apply automatically, risky ones wait for you
- **Dashboard** — web UI at /apppilot/dashboard showing status, errors, queue
- **Memory** — ELON-REPORT.md tracks everything tested, passed, failed, and trending
- **Rollback** — every change is backed up and reverted if tests fail
- **Identity Protection** — SHA-256 checksums on config files prevent tampering

## Files

| File | Purpose |
|------|---------|
| GOALS.md | Your app's mission, priorities, quality targets |
| AGENTS.md | Safe file paths and protected paths |
| SOUL.md | Agent identity and owner verification |
| ELON.md | Current constraint focus (auto-generated) |
| ELON-REPORT.md | Full test results and progress report (auto-generated) |

## Version 0.3.0 Changelog
- Added ELON strategic constraint solver
- Added Playwright site crawler
- Added continuous improvement loop
- Added ELON-REPORT.md memory and reporting
- Added dashboard buttons for ELON, Crawl, Discovery
- Fixed: init no longer overwrites host app's package.json
- Fixed: JSON parser handles markdown-wrapped responses
- Fixed: Path safety strips descriptions from patterns
- Fixed: curl in command whitelist
- Fixed: Test retry with recompilation delay
- Fixed: Smart code truncation (50KB limit)
```

## Step 6: Verify everything works

Before packaging, test:

```bash
cd ~/workspace/apppilot-package

# Check all source files exist
echo "=== Source files ==="
ls src/
ls src/subagents/
ls src/middleware/
ls src/dashboard/

echo "=== CLI commands ==="
ls bin/

echo "=== Templates ==="
ls templates/
ls templates/subagents/

echo "=== No ShopTok references ==="
grep -r "shoptok\|ShopTok\|SHOPTOK" src/ bin/ templates/ || echo "Clean — no app-specific references"

echo "=== No hardcoded paths ==="
grep -r "/home/runner" src/ bin/ || echo "Clean — no hardcoded paths"

echo "=== No leaked secrets ==="
grep -r "sk-\|api_key\|password\|secret" src/ bin/ --include="*.js" | grep -v "process.env\|apiKey\|API_KEY\|internalKey" || echo "Clean — no leaked secrets"

echo "=== Package.json version ==="
cat package.json | grep version
```

## Step 7: Pack it

```bash

ls -la apppilot-0.3.0.tgz
echo "Package size:"
du -h apppilot-0.3.0.tgz
```
