## Mission

MAinager is a Boutique Operating System that transforms how independent retail boutiques run their daily operations. It's for small business owners (1-5 locations, 3-15 employees) who are drowning in operational chaos — forgotten tasks, inconsistent procedures, no visibility into what's actually happening on the floor.

The vision: a boutique owner walks in on Monday morning and MAinager has already briefed them on yesterday's numbers, flagged that the opening checklist wasn't completed on time Saturday, suggested a staffing adjustment for the slow Tuesday ahead, and reminded them that VIP customer Sarah's birthday is Thursday. The owner spends less time managing operations and more time doing what they love — curating products and building customer relationships.

MAinager is built on six proven methodologies: 2 Second Lean (continuous improvement), Profit First (financial discipline), Remarkable Retail (customer experience), SOP Excellence (procedure consistency), GTD (task management), and AI as Copilot (cognitive load reduction).

## Architecture Context

- **Framework**: Express (backend) + React 18 with TypeScript (frontend), built with Vite
- **Database**: PostgreSQL on Neon serverless, managed with Drizzle ORM
- **Auth**: Clerk (OAuth/SSO with RBAC — Owner/Admin/Employee roles, 30+ permissions)
- **AI**: Anthropic Claude (claude-sonnet-4-20250514) for all AI features — inbox clarification, SOP surfacing, pattern recognition, Morning Whisper, staffing optimization
- **Real-time**: WebSocket server for live updates (shift changes, task assignments, notifications)
- **Integrations**: Shopify GraphQL Admin API (sales data sync, staffing recommendations), YouTube Data API v3 (private channel video management for Improvement Videos)
- **Video Storage**: YouTube private channel (primary) with AWS S3/CloudFront as fallback for video hosting
- **Hosting**: Replit (deployment target: cloudrun)
- **UI Library**: shadcn/ui + Radix UI + Tailwind CSS

## Current Priorities

Sneebly works top-down. Higher = fix first.

1. **Reliability of LIVE features** — Clock-in/out, geofencing, scheduling, payroll reports, task assignment, and Clerk auth MUST work flawlessly. These are the foundation. A broken clock-in at 9:30 AM on a Saturday is a crisis.
2. **API performance** — Every endpoint under 500ms p95. The app is used on mobile devices in retail stores with variable connectivity. Slow = unusable.
3. **Shopify sync stability** — Real-time sales data streaming is IN DEV and critical. Broken sync means the AI Staffing Optimizer, Midday Pulse, and Morning Whisper all give bad data.
4. **Error handling and graceful degradation** — If Claude API is down, the app should still function for core operations (clock-in, tasks, SOPs). AI features degrade gracefully with cached data or sensible defaults.
5. **WebSocket reliability** — Real-time updates power shift change notifications, task assignment alerts, and live dashboard data. Dropped connections must auto-reconnect.
6. **Database query performance** — As SOP, GTD, and issue tracker tables grow, queries must stay fast. Index early, paginate everything, avoid N+1 patterns.

## Feature Specifications

### Pillar 1: Employee Experience Engine

Every feature maps to at least one Self-Determination Theory need (Autonomy, Competence, Relatedness). This is intentional and non-negotiable.

#### 1.1 The Vibe System (Replaces Traditional Gamification)

Rather than points and badges, MAinager uses a "Vibe" system designed around intrinsic motivation.

- **Personal Growth Dashboard** — Each employee sees: SOP mastery progression (competence), improvement ideas submitted (autonomy), peer kudos received (relatedness). NO comparison to others by default.
- **BYB Tracker (Better Your Best)** — Tommy Mello-inspired. Tracks personal bests: fastest opening checklist, most consistent task completion, longest improvement streak. Competes against YOUR history, never others.
- **Kudos Wall** — Peer-to-peer recognition. Any employee can give a "kudo" to any other with a short message. Kudos appear in the team feed and the Morning Huddle.
- **Improvement Streak** — Track consecutive days where the employee submitted at least one improvement idea. Visible only to them. No shaming for breaks — just a gentle restart.
- **Voluntary Challenges** — Weekly opt-in challenges (never mandatory). Example: "This week: find one way to save 2 seconds in your opening routine." Autonomy-first design.

**UX Principle: Never Mandatory Fun.** Every engagement feature must be opt-in. Employees choose their challenges, choose whether to share improvements, choose whether to appear on any team view. Forced participation destroys the autonomy that makes the system work.

#### 1.2 "Fix What Bugged You?" Daily Improvement Prompt

Paul Akers-inspired continuous improvement engine.

- **End-of-Shift Prompt** — "What bugged you today?" Free text + optional photo. AI categorizes and routes to the right person.
- **Daily Improvement Quote** — Each team member's dashboard displays an AI-curated daily quote about improvement and self-improvement. Claude selects quotes that are powerful, relevant, and rotating — never the same quote twice in a 90-day window. Quotes come from lean thinkers, business leaders, athletes, philosophers. Displayed prominently as a motivational anchor on the dashboard.
- **AI Categorization** — Submissions are auto-tagged by category (process, equipment, customer experience, workspace, training) and routed to the appropriate owner/manager.
- **Improvement Trend Analysis** — AI analyzes patterns: "70% of 'What Bugged You' submissions this month relate to the fitting room. Time for an SOP revision?"
- **SOP Evolution Loop** — Employee submissions + AI insights trigger SOP revision proposals for owner review. This is how SOPs stay alive.

#### 1.3 Improvement Video Platform (Paul Akers 2 Second Lean Style)

A YouTube-style internal platform for teams to create and share short improvement videos. This is the most visible, viral piece of the continuous improvement culture.

**Core Concept:** Employees record 60-second before/after videos showing a small improvement they made. Videos are uploaded to the store's private YouTube channel (via YouTube Data API v3) and displayed in a custom YouTube-style feed inside MAinager.

**Video Creation Flow:**
1. Employee taps "Record Improvement" on their dashboard
2. In-app camera captures up to 60 seconds of video
3. Employee adds a title and short description ("I reorganized the bow supplies so we don't have to dig anymore")
4. Video uploads to the store's private YouTube channel (unlisted/private visibility)
5. MAinager stores the YouTube video ID, metadata, and employee association in the database
6. Video appears in the Improvement Feed within 30 seconds

**Improvement Feed (YouTube-Style Homepage):**
- Scrollable feed showing latest video uploads across the team
- Thumbnail grid layout with title, employee name, upload date, like count
- Filter by: Most Recent, Most Liked, By Employee, By Category
- Like button (one like per employee per video) and comment thread per video
- AI-generated "Improvement of the Week" featured banner — Claude picks the most impactful video based on likes, comments, and improvement category relevance

**Technical Approach:**
- **Primary storage**: YouTube Data API v3 — videos upload to a store-owned private YouTube channel. MAinager acts as a wrapper/client managing the channel via OAuth.
- **Fallback storage**: AWS S3 + CloudFront for stores that don't want YouTube dependency. Video files stored in S3, served via CloudFront CDN.
- **Database tables**: `improvement_videos` (id, employee_id, store_id, youtube_video_id, s3_url, title, description, category, created_at), `video_likes` (id, video_id, employee_id, created_at), `video_comments` (id, video_id, employee_id, comment_text, created_at)
- **Privacy**: All videos are private/unlisted on YouTube — only accessible through MAinager's authenticated feed. S3 videos use signed URLs with expiration.
- **Mobile-first**: Recording UI optimized for phone cameras. Landscape and portrait supported. Max file size: 100MB. Supported formats: MP4, MOV.

**Integration with Morning Huddle:** The AI pulls the most recent improvement video into the Morning Huddle agenda as the "Win of the Day" or "Improvement Spotlight."

#### 1.4 Role-Based Experience Layers

Different roles have different needs. MAinager should feel personalized from the first login.

- **Associate** — Simple, clear, encouraging. See YOUR tasks, YOUR SOPs, YOUR growth. Minimal admin noise. Feels like a helpful coach, not a surveillance tool.
- **Manager/Admin** — Operational visibility + team health. See task completion rates, SOP compliance, team kudos flow, and AI-flagged issues. Feels like a command center.
- **Owner** — Strategic overview. Morning Whisper, weekly review, financial insights, anomaly detection. Feels like having a COO who never sleeps.

#### 1.5 Emotional Design System

How the app FEELS is as important as what it does:

1. **Celebration over surveillance** — When a task is completed, the response is celebratory. Never "Task completed at 9:47 AM." Always "Opening checklist done! Nice work getting ahead of the rush."
2. **Progress over perfection** — Show how far you've come, not how far you have to go. A 60% SOP mastery score is "You've mastered 12 out of 20 procedures!" not "8 procedures incomplete."
3. **Warmth over efficiency** — MAinager's AI voice should feel like a supportive team lead, not a corporate compliance bot. Use the employee's name. Reference their recent wins.
4. **Privacy by default** — Individual performance data is visible only to the individual and their direct manager. Never broadcast individual struggles to the team.
5. **Recovery over punishment** — A missed SOP step triggers a gentle coaching moment, not a write-up. AI says: "Heads up — looks like the fitting room reset got skipped. Want me to add it to your afternoon tasks?"

### Pillar 2: The Living SOP System

SOPs in MAinager are not static documents. They are living, breathing, AI-enhanced operational intelligence that evolves with the business.

#### 2.1 SOP Architecture

- **SOP Template** — The master procedure, version-controlled. Created by owner/admin. Contains steps, quality checkpoints, expected duration, role assignments, and training notes.
- **SOP Execution** — A live instance of a template being performed by a specific employee at a specific time. Tracks step completion, time per step, notes, photos, skip reasons.
- **SOP Intelligence** — AI layer that analyzes execution data across all instances. Identifies bottlenecks, frequently skipped steps, time anomalies, and improvement opportunities.
- **SOP Evolution** — Feedback loop where employee "What Bugged You?" submissions and AI insights trigger SOP revision proposals for owner review.

#### 2.2 Context-Aware SOP Surfacing

MAinager's killer feature. SOPs appear at the right time for the right person without being asked.

**Surfacing Triggers:**
- **Time-based**: Opening checklist surfaces automatically at shift start. Closing checklist appears 30 min before close.
- **Event-based**: Shift handoff SOP surfaces when clock-in overlaps with another employee's clock-out.
- **Sales-based**: When Shopify data shows a sudden traffic spike, the "rush hour protocol" SOP surfaces.
- **Role-based**: New employee sees training-mode SOPs with extra detail. Veterans see streamlined versions.
- **Ask-based**: Employee asks MAinager AI a question, and the relevant SOP section is surfaced inline.
- **Issue-based**: When an issue is logged (e.g., "fitting room messy"), the relevant reset SOP surfaces.

#### 2.3 SOP Builder

The SOP builder should be dead simple — a boutique owner with no tech background should be able to create a professional SOP in under 5 minutes.

- **AI-Assisted Creation** — Owner describes the procedure in plain English or voice. Claude generates a structured SOP with steps, checkpoints, and estimated times. Owner reviews and edits.
- **Step Types** — Action step (do this), Verification step (confirm this is done), Photo step (take a picture), Decision step (if X then Y), Timer step (wait N minutes).
- **Quality Checkpoints** — Any step can be marked as a "checkpoint" requiring photo evidence or manager sign-off.
- **Version Control** — Every edit creates a new version. Previous versions are preserved. AI can diff versions.
- **Training Mode Toggle** — Each SOP has a "training mode" with expanded explanations, videos, and "why we do this" context.
- **Estimated Duration** — AI learns actual completion times and suggests realistic duration estimates.
- **Category Tags** — Opening, Closing, Customer Service, Visual Merchandising, Inventory, Safety, Custom.

### Pillar 3: The AI Brain — MAinager's Intelligence Layer

MAinager's AI isn't a chatbot bolted onto an operations tool. It's the connective tissue between every system, making the whole greater than the sum of its parts.

#### 3.1 "Ask MAinager" AI Copilot

Any employee can ask MAinager a question in natural language and get an answer grounded in the store's actual SOPs, policies, and data.

| Question Type | How MAinager Answers |
|---|---|
| "How do I process a return?" | Surfaces the Return Processing SOP, walks through it step by step. Asks if they want training mode. |
| "Who's working tomorrow?" | Pulls schedule data, shows tomorrow's team and their shift times. |
| "What were our sales yesterday?" | Queries Shopify sync data, provides summary with comparison to same day last week. |
| "The steamer isn't working." | Logs an issue, surfaces the Equipment Troubleshooting SOP, and notifies the manager. |
| "What should I be doing right now?" | Checks the employee's role, current time, active tasks, and incomplete SOPs. Prioritizes and suggests. |
| "Why do we fold the jeans this way?" | Pulls the "why" context from the training mode of the Visual Merchandising SOP. |

#### 3.2 AI Prompt Architecture (RAG + Context)

The AI Brain uses a Retrieval-Augmented Generation (RAG) pattern with multi-layer context:

1. **Store Context Layer**: Store name, location, timezone, business hours, current Shopify plan, owner preferences.
2. **Role Context Layer**: The asking employee's role, permissions, shift status, recent activity, SOP mastery level.
3. **Temporal Context Layer**: Current time relative to business hours, day of week, season, any active promotions or events.
4. **SOP Knowledge Layer**: Full SOP library indexed for semantic search. The AI retrieves the 3-5 most relevant SOP sections before answering.
5. **Operational Data Layer**: Today's schedule, active tasks, recent issues, Shopify sales data, staffing levels.
6. **Conversation History**: Last 10 messages in the current conversation for follow-up questions.

**Technical: Claude API Integration Pattern** — System prompt = Store Context + Role Context + Temporal Context (rebuilt per request). User message = Employee question + retrieved SOP chunks (top-k semantic search). Model: claude-sonnet-4-20250514 with 10s timeout, fallback to cached responses. Every AI response includes a confidence indicator and a "Was this helpful?" feedback loop.

#### 3.3 AI-Powered Background Insights

Beyond answering questions, the AI Brain continuously analyzes operational data to surface insights:

- **SOP Friction Detection** — "Step 4 of the Opening Checklist takes 3x longer than other steps. Consider splitting it."
- **Staffing Pattern Recognition** — "Sales peak 11am-1pm on Saturdays but you're scheduling peak staff at 2pm. Shift earlier?"
- **Task Completion Anomalies** — "Closing checklist completion dropped 40% this week. Might be related to the new hire training gap."
- **Improvement Trend Analysis** — "70% of 'What Bugged You' submissions this month relate to the fitting room. Time for an SOP revision?"
- **Predictive Scheduling** — "Based on Shopify data, next Tuesday looks slow. Consider reducing to 2 staff."

#### 3.4 AI Smart Scheduler with Shift Overlap

The AI Scheduler goes beyond basic availability matching. It builds schedules that account for operational needs including structured handoff time.

**Shift Overlap for Briefing & 3S:**
- The AI Scheduler automatically builds in a configurable overlap window (default: 1 hour) between consecutive shifts. This overlap time is dedicated to:
  - **Shift Briefing** — Outgoing employee briefs incoming employee on the day's status, open issues, customer notes, and incomplete tasks.
  - **3S (Sweep, Sort, Standardize)** — Paul Akers-inspired. The first activity of every shift. Not cleanup — it's teaching employees to SEE waste. Both outgoing and incoming employees participate.
  - **Morning Huddle** (if it's the first shift of the day) — The overlap window accommodates the guided standup without cutting into selling time.
- **Configurable per store**: Owner sets overlap duration (30 min, 45 min, 1 hour) in store settings. AI respects this when generating schedule suggestions.
- **Smart overlap**: AI only schedules overlap when there IS a preceding shift. First shift of the day gets extra pre-open time instead. AI factors in labor cost impact and alerts the owner if overlap hours push payroll above budget thresholds.
- **Handoff SOP auto-surfaces**: When the overlap window begins (detected by overlapping clock-ins), the Shift Handoff SOP automatically surfaces for both employees.

### Pillar 4: The Daily Ritual System

Inspired by Paul Akers' daily morning meeting, MAinager's daily rituals create a connected narrative arc for every workday. This is what turns a tool into a culture.

#### 4.1 The MAinager Day: Connected Ritual Flow

| Time | Ritual |
|---|---|
| Pre-shift (Auto) | **Morning Whisper** — AI generates a brief audio/text briefing for the owner: yesterday's numbers, today's schedule, flagged issues, notable events. |
| Shift Start (5 min) | **Morning Huddle Mode** — Guided standup: review yesterday's wins, today's goals, any open issues. Includes one 2 Second Lean principle of the day. |
| Opening (10-15 min) | **Opening Checklist SOP** — Step-by-step with quality checkpoints. Auto-surfaces at shift start. |
| Midday (Auto) | **Midday Pulse** — AI checks Shopify sales vs. target, flags if ahead or behind, suggests adjustments. |
| Shift Change | **Shift Handoff Protocol** — Outgoing employee completes handoff SOP during overlap window. Incoming employee reviews notes. Both participate in 3S. |
| Pre-close (30 min) | **Closing Checklist SOP** — Step-by-step closing procedures with quality checkpoints. |
| End of Day (2 min) | **Daily Debrief Capture** — Quick reflection: What went well? What bugged you? Any customer highlights? + Improvement video prompt. |
| Friday 3pm (Owner) | **Weekly Review Ritual** — AI-guided GTD review. Process inbox, review projects, check Waiting For items, update Someday/Maybe list. |

#### 4.2 Morning Huddle Mode

The Morning Huddle is the heartbeat of MAinager's culture system, directly inspired by FastCap's daily meeting. It should take 5-10 minutes and leave the team energized.

**Huddle Agenda (auto-generated by AI):**

1. **Win of the Day** — AI selects yesterday's best moment: a great sale, a completed SOP improvement, a customer compliment, or the latest improvement video. Celebrates the team.
2. **Lean Principle** — Rotates through 20 core principles (from 2 Second Lean). Short, memorable, actionable. Example: "Action This Day — if you see something that needs fixing, fix it NOW."
3. **Today's Goals** — AI pulls from: scheduled tasks, open issues, SOP deadlines, sales targets. Presents 3-5 clear goals.
4. **Heads Up** — Anything the team needs to know: new products arriving, VIP customer visiting, schedule change, weather impact.
5. **Quick Kudos** — Surface any peer kudos from the last 24 hours. Quick acknowledgment.

**Design Note: The Huddle is NOT a Status Meeting.** Status meetings drain energy. The Morning Huddle is designed to GIVE energy. It's short, celebratory, forward-looking, and principle-driven. Think sports team pre-game, not corporate standup.

## Technical Standards

- All database queries must use Drizzle ORM with parameterized inputs
- All API responses must follow the shape: `{ success: boolean, data?: T, error?: { message: string, code: string } }`
- All new routes need Zod input validation middleware
- All WebSocket events must include a `type` field and follow existing naming conventions
- All Claude API calls must include timeout handling (10s default, 30s for Morning Whisper generation)
- All Claude API calls must have fallback behavior if the API is unavailable (cached response, sensible default, or graceful error message)
- All Shopify API calls must handle rate limiting (check `X-Shopify-Shop-Api-Call-Limit` headers) and retry with exponential backoff
- All YouTube API calls must handle quota limits (10,000 units/day default) and degrade gracefully when quota is exhausted
- All time-related logic must respect the store's configured timezone (not UTC, not server time)
- All payroll calculations must use decimal arithmetic (never floating point) — use a library like `decimal.js` or integer cents
- All user-facing timestamps must be formatted in the store's locale and timezone
- Clerk middleware must be applied to all protected routes — never bypass auth checks
- RBAC checks must happen at the route level AND the service level (defense in depth)
- All database transactions must be used for multi-step operations (e.g., completing an SOP execution should atomically update `sop_executions` and `activity_log`)
- Never store derived data that can be computed from source data (except for performance-critical caches with TTLs)
- All environment variables must be accessed through a centralized config module, never `process.env` directly in route handlers
- All video uploads must validate file type (MP4, MOV), file size (max 100MB), and duration (max 60 seconds) before upload
- All video content must be private/unlisted — never publicly accessible outside authenticated MAinager sessions
