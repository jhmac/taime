Build the Midday Pulse (automated noon sales check) and the Shift Handoff Protocol SOP.

BACKEND:

1. MIDDAY PULSE — Create `server/services/middayPulse.ts`:

   Function: generateMiddayPulse(storeId: string): Promise<MiddayPulseData>
   
   - Query Shopify sync data for today's sales so far
   - Compare to: same day last week, daily sales target (if configured in store settings)
   - Calculate: total revenue, transaction count, average order value, pace to hit daily target
   
   Call Claude to generate a brief pulse summary:
   System prompt:
   """
   You are MAinager's Midday Pulse. Generate a brief, encouraging midday sales check-in for a retail boutique team.
   
   Tone: honest but encouraging. If sales are behind, frame it as an opportunity, not a failure. If ahead, celebrate.
   
   Return JSON:
   {
     "headline": "One punchy line summarizing the day so far (max 15 words)",
     "detail": "2-3 sentences with specific numbers and context",
     "suggestion": "One actionable suggestion based on the data (optional, only if genuinely helpful)"
   }
   """
   
   - 10s timeout, fallback to raw numbers without AI commentary
   - Cache the result (one pulse per store per day)

   Scheduled job: Run at 12:00 PM (noon) store timezone
   - Generate pulse
   - Send WebSocket notification to all on-shift employees: { type: 'midday_pulse', data: pulseData }
   - Store in a `midday_pulses` table (create this table: id, store_id, pulse_date, data jsonb, created_at)

2. API endpoint:
   - GET /api/rituals/pulse/today — Returns today's midday pulse (generates on-demand if after noon and not cached)

3. SHIFT HANDOFF PROTOCOL — Create a default SOP template:

   When a store is first set up (or via a seed/migration), auto-create a "Shift Handoff Protocol" SOP template:
   
   Title: "Shift Handoff Protocol"
   Category: "shift_handoff"
   Estimated duration: 15 minutes
   Training notes: "A smooth handoff means nothing falls through the cracks. The outgoing team member shares what happened, what's open, and what to watch for. The incoming team member asks questions and starts with 3S."
   
   Steps:
   1. (Action) "Review open issues" — "Check the issue tracker for any unresolved issues from this shift"
   2. (Action) "Handoff notes" — "Tell your teammate: what happened today, any customer situations, anything that needs attention"
   3. (Verification) "Check task board" — "Review the task list — are there incomplete tasks to pass along?"
   4. (Action) "3S: Sweep" — "Walk the store together. What's out of place?"
   5. (Action) "3S: Sort" — "Put things where they belong. Discard what doesn't"
   6. (Action) "3S: Standardize" — "Is everything set up the standard way? If not, fix it now"
   7. (Verification) "Ready to go" — "Incoming team member confirms they're briefed and ready"

   Create this via a database seed or a setup function that runs when the SOP module is first activated.

FRONTEND:

4. MiddayPulseCard (dashboard component):
   - Shows after noon on the employee dashboard
   - Card with: headline (bold), detail text, suggestion (if present)
   - Shopify data summary: revenue so far, transaction count, vs target or vs last week
   - Compact, informative, not overwhelming
   - Auto-appears via WebSocket push at noon

5. MiddayPulseNotification:
   - When the WebSocket event arrives, show an in-app notification/toast
   - "📊 Midday Pulse: [headline]" — tap to expand

TECHNICAL:
- Midday Pulse only runs if Shopify sync is configured and has recent data
- If Shopify data is stale (>2 hours old), show a warning: "Sales data may be delayed"
- The handoff SOP is a regular SOP template — it uses all the existing SOP infrastructure
