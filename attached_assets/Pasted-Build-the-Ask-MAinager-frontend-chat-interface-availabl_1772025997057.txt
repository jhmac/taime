Build the "Ask MAinager" frontend chat interface — available to every employee from anywhere in the app.

LOCATION: Create components in `client/src/features/ai-copilot/`

1. ASK MAINAGER BUTTON (global, persistent):
   
   Add a floating "Ask MAinager" button to the main app layout:
   - Fixed position: bottom-right corner (if the GTD capture button is there too, stack them — MAinager above, capture below)
   - Circular button with a sparkle/brain/chat icon and "Ask" label
   - Subtle pulse animation when idle (draws attention but not annoying — stops after 3 seconds)
   - Tap → opens AskMAinagerSheet

2. AskMAinagerSheet (slide-up sheet on mobile, side panel on desktop):
   
   Chat-style interface:
   
   Header:
   - "MAinager" with a friendly avatar/icon
   - Subtitle: "Your AI assistant" or context-aware: "Good morning, [name]! How can I help?"
   - Close button (X)
   - Minimize button (collapses to the floating button)
   
   Chat Area:
   - Message bubbles:
     - User messages: right-aligned, colored
     - MAinager responses: left-aligned, light background, with the MAinager avatar
     - Typing indicator while waiting for response: "MAinager is thinking..."
   - Suggested Actions:
     - When the AI returns suggested_actions, render them as tappable buttons below the message:
       - "Start SOP: [Title]" → navigates to SOP execution
       - "Log Issue" → opens the issue reporter
       - "View Task" → navigates to the task
       - "View Schedule" → navigates to schedule page
     - These buttons are inline with the chat, visually distinct (outlined, accent color)
   - Referenced SOPs:
     - If the AI references SOPs, show them as linked chips below the answer
     - Tap → opens the SOP detail page
   - Confidence indicator:
     - High confidence: no indicator (default)
     - Medium confidence: subtle note: "I'm fairly sure about this, but you may want to double-check."
     - Low confidence: note: "I'm not sure about this — the answer may not be in our procedures yet."
   - Feedback buttons:
     - Below each AI response: 👍 / 👎 (small, unobtrusive)
     - On 👎: optionally expand a text field: "What was wrong?" → POST /api/ai/feedback
   
   Input Area:
   - Text input at bottom, auto-focused when sheet opens
   - Placeholder: "Ask me anything about the store..."
   - Send button (or Enter to send)
   - Quick suggestion chips above the input (context-aware):
     - If it's morning: "What should I do first?" / "Who's working today?"
     - If mid-day: "How are sales today?" / "Any open issues?"
     - If near closing: "What's left on the closing checklist?"
     - Always available: "How do I..." (triggers a second line: "process a return?", "steam clothes?", etc.)

3. CONVERSATION HISTORY:
   - Messages persist within the sheet during the session
   - "New Conversation" button in the header to clear and start fresh
   - Previous messages scroll up, new messages appear at bottom with auto-scroll
   - Conversation resets daily (new context each day)

4. EMPTY STATE (first time / new conversation):
   - Friendly greeting: "Hey [name]! I'm MAinager, your AI assistant. I know all about [store name]'s procedures, schedules, and sales data. Ask me anything!"
   - Quick suggestion grid:
     - "What should I be doing?"
     - "How are sales today?"
     - "Help me find a procedure"
     - "Report a problem"

5. KEYBOARD & MOBILE OPTIMIZATION:
   - On mobile: sheet takes full screen height minus a peek area
   - Input stays above the keyboard (position: fixed adjustments)
   - Large send button for one-handed use
   - Swipe down to minimize

DESIGN:
- The chat should feel like texting a knowledgeable, supportive coworker
- Responses should render almost instantly (optimistic "thinking" indicator)
- The UI never blocks the rest of the app — employee can minimize and come back
- No corporate chatbot aesthetic — warm, personal, boutique-appropriate

TECHNICAL:
- Use the POST /api/ai/ask endpoint
- Maintain conversation_id in state for follow-up questions
- Auto-retry once on timeout (with "Taking a moment..." message)
- Conversation messages stored in React state (not localStorage — conversations are ephemeral)
- The sheet component must handle keyboard avoidance on mobile properly
- TypeScript strict: full types for all AI response structures
