Build the semantic search infrastructure that powers the "Ask MAinager" AI Copilot. This is the Retrieval-Augmented Generation (RAG) foundation — it indexes all SOP content for semantic search so the AI can find relevant procedures when answering questions.

STEP 1: EMBEDDING GENERATION SERVICE

Create `server/services/embeddingService.ts`:

We need to convert SOP text content into vector embeddings for semantic search. Use one of these approaches (in order of preference):

Option A — Anthropic's Claude for embeddings (if available):
- Check if the Anthropic API supports embeddings. If not, use Option B.

Option B — OpenAI-compatible embedding API:
- Use a lightweight embedding model. Install `openai` package.
- Use model: "text-embedding-3-small" (cheap, fast, 1536 dimensions)
- OR if you want to avoid an OpenAI dependency, use Option C.

Option C — Local embeddings (no external API):
- Install `@xenova/transformers` (runs transformer models in Node.js)
- Use model: 'Xenova/all-MiniLM-L6-v2' (384 dimensions, runs locally, no API key needed)
- This is the most self-contained option for a boutique app

Function: generateEmbedding(text: string): Promise<number[]>
- Takes text, returns a vector embedding
- Handles errors gracefully, logs latency
- Cache embeddings to avoid regenerating for unchanged content

STEP 2: VECTOR STORAGE

We need to store and search embeddings. Two approaches:

Option A — pgvector extension (preferred if Neon supports it):
- Check if pgvector is available on Neon: run `CREATE EXTENSION IF NOT EXISTS vector;`
- If available, use it.

Option B — Simple cosine similarity in application code:
- Store embeddings as jsonb arrays in PostgreSQL
- Compute cosine similarity in a service function
- Slower but works everywhere without extensions

CREATE A NEW MIGRATION:

TABLE: sop_embeddings
- id: uuid, primary key, default gen_random_uuid()
- store_id: text, not null, references stores(id)
- source_type: text, not null (enum-like: 'sop_template', 'sop_step', 'sop_training_notes')
- source_id: uuid, not null (ID of the sop_template or sop_step)
- content_text: text, not null (the text that was embedded — for display in search results)
- content_hash: text, not null (hash of content_text — to detect when re-embedding is needed)
- embedding: vector(384) if using pgvector, OR jsonb if not (the embedding vector)
- created_at: timestamp with timezone, default now()
- updated_at: timestamp with timezone, default now()

If using pgvector:
- CREATE INDEX ON sop_embeddings USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);

INDEX: sop_embeddings(store_id, source_type)
INDEX: sop_embeddings(source_id)
UNIQUE: sop_embeddings(source_id, source_type)

STEP 3: INDEXING SERVICE

Create `server/services/sopIndexer.ts`:

Function: indexSOPTemplate(templateId: string): Promise<void>
- Fetches the SOP template and all its steps
- Creates embeddings for:
  a. Template title + description (source_type: 'sop_template')
  b. Each step's title + description combined (source_type: 'sop_step')
  c. Template training_notes if present (source_type: 'sop_training_notes')
  d. Each step's training_detail if present (source_type: 'sop_step' — appended to step text)
- Checks content_hash to skip unchanged content
- Upserts into sop_embeddings

Function: indexAllSOPs(storeId: string): Promise<{ indexed: number, skipped: number }>
- Indexes all active SOP templates for a store
- Used for initial setup and periodic re-indexing

Function: searchSOPs(storeId: string, query: string, topK: number = 5): Promise<SOPSearchResult[]>
- Generates an embedding for the query
- Finds the topK most similar SOP embeddings for this store
- Returns: [{ source_type, source_id, content_text, similarity_score, template_id, template_title }]
- If using pgvector: use `<=>` operator for cosine distance
- If using jsonb: compute cosine similarity in JS and sort

STEP 4: AUTO-INDEXING TRIGGERS
- When an SOP template is created or updated (in the SOP API from Phase 1): queue re-indexing
- Create a cron job that runs nightly to re-index any SOPs that changed since last index
- On initial setup: provide a one-time index-all endpoint: POST /api/sops/reindex (Owner only)

STEP 5: TESTING
- After building, create a test SOP about "processing returns" and another about "opening the store"
- Index them
- Search for "how do I handle a return" and verify the return SOP comes back as the top result
- Search for "what do I do first thing in the morning" and verify the opening SOP ranks highest

TECHNICAL:
- Embedding generation should be async and non-blocking for the main request flow
- If the embedding API/model is unavailable, log a warning and skip (don't crash)
- Rate limit embedding generation to avoid API quota issues
- All embedding operations logged with structured logger
