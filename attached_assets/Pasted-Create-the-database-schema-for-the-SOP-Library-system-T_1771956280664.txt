Create the database schema for the SOP Library system. This is the foundation for checklists, procedures, and all operational intelligence in MAinager.

CREATE A NEW MIGRATION with these tables:

TABLE: sop_templates
- id: uuid, primary key, default gen_random_uuid()
- store_id: text, not null, references stores(id)
- title: text, not null (e.g., "Opening Checklist", "Return Processing")
- description: text, nullable (brief summary of what this SOP covers)
- category: text, not null (enum-like: 'opening', 'closing', 'customer_service', 'visual_merchandising', 'inventory', 'safety', 'shift_handoff', 'custom')
- estimated_duration_minutes: integer, nullable (AI will learn and suggest this)
- role_assignments: jsonb, nullable (array of role strings this SOP applies to: ['associate', 'manager', 'owner'])
- is_active: boolean, default true
- training_notes: text, nullable (expanded "why we do this" context for training mode)
- version: integer, not null, default 1
- parent_template_id: uuid, nullable, self-references sop_templates(id) (for version history — points to the previous version)
- created_by: text, not null (Clerk user ID)
- created_at: timestamp with timezone, default now()
- updated_at: timestamp with timezone, default now()

TABLE: sop_steps
- id: uuid, primary key, default gen_random_uuid()
- template_id: uuid, not null, references sop_templates(id) ON DELETE CASCADE
- step_order: integer, not null (1-based ordering)
- title: text, not null (short action description: "Vacuum zone 1")
- description: text, nullable (detailed instructions)
- step_type: text, not null (enum-like: 'action', 'verification', 'photo', 'decision', 'timer')
- is_checkpoint: boolean, default false (requires photo evidence or manager sign-off)
- timer_duration_seconds: integer, nullable (only for 'timer' step type)
- decision_options: jsonb, nullable (only for 'decision' type — { "options": [{ "label": "Yes", "next_step_order": 5 }, { "label": "No", "next_step_order": 6 }] })
- training_detail: text, nullable (expanded explanation shown in training mode)
- created_at: timestamp with timezone, default now()

TABLE: sop_executions
- id: uuid, primary key, default gen_random_uuid()
- template_id: uuid, not null, references sop_templates(id)
- employee_id: text, not null (Clerk user ID)
- store_id: text, not null, references stores(id)
- status: text, not null, default 'in_progress' (enum-like: 'in_progress', 'completed', 'abandoned', 'paused')
- started_at: timestamp with timezone, default now()
- completed_at: timestamp with timezone, nullable
- notes: text, nullable (overall execution notes)
- created_at: timestamp with timezone, default now()

TABLE: sop_step_completions
- id: uuid, primary key, default gen_random_uuid()
- execution_id: uuid, not null, references sop_executions(id) ON DELETE CASCADE
- step_id: uuid, not null, references sop_steps(id)
- status: text, not null, default 'pending' (enum-like: 'pending', 'completed', 'skipped')
- completed_at: timestamp with timezone, nullable
- time_spent_seconds: integer, nullable (tracked automatically)
- skip_reason: text, nullable (required if status is 'skipped')
- photo_url: text, nullable (for photo steps and checkpoints)
- notes: text, nullable
- manager_sign_off: boolean, default false
- manager_sign_off_by: text, nullable (Clerk user ID of manager who signed off)
- manager_sign_off_at: timestamp with timezone, nullable
- created_at: timestamp with timezone, default now()

INDEXES (create these in the same migration):
- sop_templates(store_id, is_active, category)
- sop_templates(store_id, created_at)
- sop_steps(template_id, step_order)
- sop_executions(store_id, employee_id, status)
- sop_executions(template_id, started_at)
- sop_executions(store_id, started_at)
- sop_step_completions(execution_id, step_id)
