Build the AI services that power the Morning Huddle content generation and the Daily Improvement Quote.

BACKEND:

1. Create `server/services/morningHuddleAI.ts`:

Function: generateHuddleContent(storeId: string, date: Date): Promise<HuddleContent>

This function gathers data and calls Claude to generate the huddle agenda.

DATA GATHERING (before the AI call):
- Yesterday's sales from Shopify sync data (total revenue, transaction count, top selling item)
- Yesterday's completed SOP executions (count, any notable completions)
- Yesterday's resolved issues (count, any notable ones)
- Today's scheduled employees and shift times
- Today's open/overdue tasks
- Any open issues with 'urgent' or 'high' priority
- Kudos from the last 24 hours
- Latest improvement video (if the Improvement Video feature is built — otherwise skip)

CLAUDE CALL:
System prompt:
"""
You are MAinager, a warm and energizing AI assistant for a retail boutique team. Generate the Morning Huddle agenda for today.

Your tone is: enthusiastic but genuine, like the best team lead who actually cares. Short sentences. Action-oriented. Never corporate-speak.

Return a JSON object:
{
  "win_of_the_day": "One specific, celebratory highlight from yesterday. Be specific — use names, numbers, details. Max 2 sentences.",
  "lean_principle": "One 2 Second Lean principle for today. Format: 'PRINCIPLE NAME — one sentence explanation and how to apply it today.'",
  "goals": ["3-5 specific, actionable goals for today based on the data provided. Each goal is one clear sentence."],
  "heads_up": ["Any alerts the team should know about. Only include if there's something genuinely worth mentioning. Can be empty."],
  "kudos_summary": "If there are kudos from the last 24 hours, summarize them warmly in 1-2 sentences. If none, omit this field."
}

Lean principles to rotate through (use a different one each day):
1. Action This Day — if you see something that needs fixing, fix it NOW.
2. Fix What Bugs You — the simplest improvement framework. What annoyed you? Fix it.
3. 2 Second Lean — find one way to save 2 seconds. Tiny improvements compound.
4. Sweep Sort Standardize — see the workspace with fresh eyes every day.
5. Make It Visual — if you improved something, show the before and after.
6. One Piece Flow — finish one task completely before starting the next.
7. Standard Work — the best known way to do something, until someone finds a better way.
8. Grow People First — lean is always spelled P-E-O-P-L-E.
9. Go See — don't assume, go look at the actual situation.
10. Respect for People — every person's idea matters, every person's time matters.
11. PDCA — Plan, Do, Check, Adjust. Try something small, see if it works, adjust.
12. Waste Walk — walk the store looking for the 8 wastes: defects, overproduction, waiting, unused talent, transportation, inventory, motion, extra processing.
13. 5 Why — when something goes wrong, ask "why?" five times to find the root cause.
14. Gemba — the real answers are where the work happens, not in a spreadsheet.
15. Kaizen — change for the better. Every single day.
16. Visual Management — organize so anyone can see what's normal and what's not in 5 seconds.
17. Pull Don't Push — respond to real demand, don't guess.
18. Flow — remove the blockers that make work stop and start.
19. Jidoka — if something is wrong, stop and fix it now, not later.
20. Hansei — honest self-reflection. What can I personally do better tomorrow?
"""

User message includes all the gathered data formatted clearly.

- 10 second timeout
- Fallback: if Claude fails, generate a basic huddle with static content ("Good morning team! Let's make today great." + today's goals from the task list)
- Cache the result in the morning_huddles table

2. Create `server/services/dailyQuoteAI.ts`:

Function: generateDailyQuote(storeId: string, date: Date): Promise<{ quote_text: string, quote_author: string }>

- First, check daily_quote_history for this store to get hashes of quotes used in the last 90 days
- Call Claude with system prompt:

"""
Generate one powerful, inspiring quote about improvement, self-improvement, continuous growth, or excellence in craft. 

The quote should resonate with retail team members — people who work hard on their feet, take pride in creating beautiful store experiences, and are learning to see waste and fix problems.

Sources can include: lean thinkers (Taiichi Ohno, Paul Akers, W. Edwards Deming), business leaders, athletes, philosophers, authors, or your own original wisdom.

Return JSON: { "quote_text": "the quote", "quote_author": "attribution" }

Do NOT use any of these previously used quotes (identified by hash):
[list of hashes]
"""

- Hash the returned quote and check it hasn't been used — if it has, retry once
- Store in daily_quotes and daily_quote_history
- Fallback: maintain a hardcoded list of 30 great quotes as backup if Claude fails

3. Create scheduled jobs (use node-cron or the existing scheduling mechanism):
   - Generate Morning Huddle content at 6:00 AM store timezone daily (so it's ready before anyone arrives)
   - Generate Daily Quote at 5:00 AM store timezone daily

4. API endpoints:
   - GET /api/rituals/huddle/today — Returns today's huddle content (generates on-demand if not cached)
   - PUT /api/rituals/huddle/today — Update huddle (mark attendees, update status)
   - GET /api/rituals/quote/today — Returns today's quote
   - POST /api/rituals/debrief — Submit daily debrief (employee submits their end-of-day reflection)
   - GET /api/rituals/debrief?date=YYYY-MM-DD — Get debriefs for a date (manager/owner sees all, employee sees their own)
   - POST /api/kudos — Send a kudo
   - GET /api/kudos?days=7 — Get recent kudos

TECHNICAL:
- All Claude calls: 10s timeout, try/catch, structured logging
- All endpoints: Clerk auth, RBAC, Zod validation
- Store timezone: use the store's configured timezone for all date calculations
