Build the GTD Inbox frontend — the capture and processing interface.

LOCATION: Create components in `client/src/features/gtd/`

1. QUICK CAPTURE — Universal Inbox Entry (available from ANYWHERE in the app)
   
   Add a persistent "Capture" floating action button to the app's main layout:
   - Small round button with a "+" icon, fixed position bottom-right (above any existing FABs — stack them)
   - Tap → opens QuickCaptureSheet (slide-up bottom sheet on mobile, popover on desktop)
   - QuickCaptureSheet:
     - Single text input, auto-focused, placeholder: "What's on your mind?"
     - Large text area for longer captures
     - "Capture" button (or Enter to submit)
     - No category, no priority, no nothing — pure zero-friction capture. Just text and go.
     - On submit: POST /api/gtd/inbox, show brief success toast "Captured! ✓", close sheet
     - The input clears and the sheet stays open for rapid-fire captures (user taps Capture again for the next one)
     - "Done" button or swipe-down to close

   This must be THE fastest interaction in the entire app. Tap → type → capture → done in under 5 seconds.

2. INBOX PAGE (`/gtd/inbox`)
   
   Shows all unprocessed and clarified inbox items.
   
   Layout:
   - Top bar: "Inbox" title + count badge + "Capture" button
   - List of inbox items, sorted newest first
   - Each item card shows:
     - Raw input text (the thing they captured)
     - Source badge (manual, debrief, issue, SOP feedback)
     - Time captured (relative: "2h ago")
     - Status indicator:
       - 🔵 Unprocessed (no AI clarification yet — show subtle "Processing..." spinner)
       - 🟢 Clarified (AI has a suggestion — ready to process)
       - ✅ Processed (done — hidden by default, toggle to show)
   
   PROCESSING FLOW (when user taps a clarified item):
   - Expand the item inline or open a ProcessInboxItemSheet
   - Show the AI's suggestion prominently:
     - "MAinager suggests: [destination]" with confidence indicator
     - Pre-filled fields based on AI suggestion: title, context, priority, time estimate, etc.
     - If AI suggests a two-minute action: highlight with "⚡ Quick win — do this now!"
   - User can:
     - Accept the suggestion (one tap: "Looks right ✓") → processes with AI defaults
     - Override any field (edit title, change destination, adjust priority, etc.)
     - Change destination entirely (dropdown: Next Action, Project, Waiting For, Someday/Maybe, Reference, Issue, Trash)
     - Each destination shows relevant fields:
       - Next Action: context, energy, time, priority, due date, assign to, link to project
       - Project: desired outcome, due date
       - Waiting For: waiting on (text or employee picker), follow-up date
       - Someday/Maybe: category
       - Reference: tags
       - Issue: routes to Issue Tracker creation
       - Trash: just archives it
   - "Process" button → calls POST /api/gtd/inbox/:id/process → item disappears from inbox
   - Success: "Processed! Added to [destination]."

3. INBOX ZERO CELEBRATION:
   - When the inbox is empty, show a warm celebration: "Inbox Zero! 🧘 Your mind is clear."
   - This is a GTD milestone — make it feel good

DESIGN:
- The inbox should feel like a trusted capture system — nothing gets lost
- Processing should feel satisfying, like clearing out clutter
- AI suggestions reduce processing to ONE TAP for most items
- Mobile-first: one-handed operation, large targets
- Dark mode support if the app has it

TECHNICAL:
- Real-time updates via WebSocket when AI clarification completes (item transitions from unprocessed → clarified)
- Optimistic UI for processing (remove item immediately, revert if API fails)
- Skeleton loading states
- Pull-to-refresh on mobile
