Rebuild/enhance the main dashboard to be role-specific. Each role (Associate, Manager, Owner) gets a different experience.

FRONTEND:

Create `client/src/features/dashboard/` with role-specific dashboard components.

1. ASSOCIATE DASHBOARD (`/dashboard` when role = employee):
   Layout (top to bottom, single column, mobile-first):
   
   a. Daily Quote Card — today's improvement quote (prominent, top of page)
   b. Surfaced SOP Banner — if any SOPs are surfaced for them right now
   c. My Tasks Today — list of assigned tasks with checkboxes
   d. My Active SOP — if they have an in-progress SOP execution, show a "Continue" button
   e. Quick Actions row:
      - "Report Issue" button
      - "Record Improvement" button (video)
      - "Give a Kudo" button
   f. Recent Improvements — 3 latest team improvement videos (thumbnail strip)
   g. My Stats (compact): tasks completed today, SOP executions this week, current improvement streak

   Emotional design: warm greeting "Good [morning/afternoon], [name]! Here's your day." 
   No admin noise, no data they don't need. Feels like a helpful coach.

2. MANAGER DASHBOARD (`/dashboard` when role = admin):
   Layout:
   
   a. Morning Huddle Card — "Start Morning Huddle" button (if first shift and huddle not done)
   b. Team On Shift — who's clocked in right now with their shift times
   c. Open Issues — count by priority, click to expand
   d. SOP Completion Today — progress bar: X of Y checklists completed
   e. Tasks Overview — overdue tasks (red), due today, upcoming
   f. Sign-Off Requests — any SOP checkpoints waiting for manager approval
   g. Team Kudos (last 24h) — quick feed
   h. Midday Pulse Card (after noon)
   i. Recent Improvement Videos — latest 3 from the team

   Feels like a command center — operational visibility without overwhelm.

3. OWNER DASHBOARD (`/dashboard` when role = owner):
   Layout:
   
   a. Morning Whisper Summary — yesterday's key numbers + today's outlook (placeholder for now — full Morning Whisper is Q3, but show what data we have)
   b. Sales Snapshot — today's revenue vs target, vs same day last week (from Shopify sync)
   c. Team Health — who's on shift, attendance today, any issues
   d. Operational Scorecard:
      - SOP completion rate this week (%)
      - Issue resolution time (avg)
      - Task completion rate (%)
      - Improvement videos this week (count)
   e. Flagged Items — anything that needs owner attention (urgent issues, budget warnings, overdue tasks)
   f. Quick Links — schedule, payroll, SOP library, issue tracker

   Feels like having a COO. Strategic overview, not task-level detail.

4. DASHBOARD ROUTING:
   - The `/dashboard` route checks the user's Clerk role and renders the appropriate dashboard
   - Use a DashboardRouter component that switches based on role
   - If role can't be determined, default to Associate view

TECHNICAL:
- Each dashboard makes its own API calls for its specific data needs
- Use React Query with appropriate cache/stale times
- Loading skeletons for each section
- Error boundaries per section (one section failing shouldn't break the whole dashboard)
- The greeting uses the store's timezone to determine morning/afternoon/evening
- Real-time updates via WebSocket for: new issues, task completions, SOP completions, new videos
